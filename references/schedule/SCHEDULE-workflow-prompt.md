# Life Index 定时任务配置工作流

> **本文档角色**: 引导Agent自主配置定时任务的交互式Prompt
> **执行方式**: Agent按步骤1-6逐一执行，每步完成后确认再进入下一步
> **核心约束**: 不依赖上下文、指令自包含、平台自适应

---

## Step 1: 理解意图（Why & What）

请先阅读并理解以下内容，确认你明白了Life Index定时任务的目的。

### 1.1 Life Index是什么

Life Index是一个**个人生活日志记录系统**，帮助用户：
- 用自然语言记录日常生活、思考、工作
- 自动索引，支持多维度检索
- 定期回顾人生轨迹

### 1.2 为什么需要定时任务

作为Agent，你在用户主动对话时可以协助记录。但以下场景**需要定时任务**：

| 场景 | 用户行为 | 定时任务价值 |
|------|---------|-------------|
| 晚间回顾 | 用户可能忘记查看今日记录 | 22:00自动推送日报，提醒回顾 |
| 周末总结 | 用户周末可能不打开应用 | 周日自动推送周报，持续追踪 |
| 月末归档 | 用户可能忘记生成月度摘要 | 月末自动生成并保存月报文件 |
| 搜索性能 | 用户无感，但需要维护 | 深夜自动维护索引，保持搜索速度 |

**核心目标**: 成为用户的"数字生活管家"，在不打扰的前提下提供价值。

### 1.3 定时任务清单

| 任务 | 触发时间 | 工作内容 | 输出形式 |
|------|---------|---------|---------|
| **日报** | 每天22:00 | 汇总今日日志，生成摘要和建议 | 推送消息 |
| **周报** | 每周日22:10 | 总结本周轨迹，含国际热点 | 推送消息 |
| **月报** | 每月末18:30 | 深度回顾，生成月度摘要文件 | 文件+推送 |
| **年报** | 12月31日19:15 | 年度人生总结 | 文件+推送 |
| **索引维护** | 每天23:50 | 更新FTS和向量索引 | 静默执行 |

### 1.4 确认理解

请向用户确认：
```
我已理解Life Index定时任务的目的：
- 每天22:00提醒用户回顾今日（日报）
- 每周日总结一周生活（周报）
- 每月末生成月度摘要文件（月报）
- 每天深夜维护搜索索引（索引维护）

这些任务将在你设定的时间自动执行，无需你主动请求。
```

**等待用户确认后再进入Step 2。**

## Step 2: 查看参考案例

以下是基于OpenClaw平台设计的定时任务模板。**注意：这是参考模板，你需要在Step 3-4中验证格式是否正确。**

### 2.1 日报任务模板

**任务名称**: `life-index-daily-report`

**Cron表达式**: `0 22 * * *`

**执行指令**:
```markdown
# Life Index 日报生成任务

## 身份与环境
你是Life Index技能的定时任务执行器。
技能目录: {{SKILL_DIR}}
今日日期: {{TODAY}}

## 执行步骤

Step 1: 获取今日日志
cd {{SKILL_DIR}}
python tools/search_journals.py --date {{TODAY}} --limit 100

Step 2: 分析结果
如果返回结果为空（无今日日志）：
  → 输出: "今日（{{TODAY}}）暂无日志记录。建议用户记录今日生活点滴。"
  → 任务结束

如果有日志：
  → 继续Step 3

Step 3: 生成日报内容
基于获取的日志，生成以下结构的日报：

【Life Index 日报】{{TODAY}}

📝 今日概要
[用2-3句话总结今日核心内容]

⚡ 要点速览
• [要点1，不超过15字]
• [要点2，不超过15字]
• [要点3，不超过15字]

💡 AI建议  
→ [建议1，祈使语气]
→ [建议2，祈使语气]

🔔 要事提醒
[待跟进事项，或"无待跟进事项"]

Step 4: 推送报告
将生成的日报内容推送给用户。

## 约束
- 总长度控制在200 tokens以内
- 如果执行失败，记录错误但不重复尝试
```

### 2.2 周报任务模板

**任务名称**: `life-index-weekly-report`

**Cron表达式**: `10 22 * * 0`

**执行指令**:
```markdown
# Life Index 周报生成任务

## 身份与环境
你是Life Index技能的定时任务执行器。
技能目录: {{SKILL_DIR}}
本周日期范围: {{WEEK_START}} 至 {{WEEK_END}}

## 执行步骤

Step 1: 获取本周日志
cd {{SKILL_DIR}}
python tools/search_journals.py --date-from {{WEEK_START}} --date-to {{WEEK_END}} --limit 500

Step 2: 生成周报
基于本周日志，生成以下结构：

【Life Index 周报】{{WEEK_START}} ~ {{WEEK_END}}

📊 本周概览
本周共记录N篇日志，涵盖[主题分布]

🎯 核心主题
[2-3个核心主题，每个1句话说明]

⚡ 高光时刻
• [本周重要记录1]
• [本周重要记录2]

📈 趋势观察
[AI对用户本周状态的观察和建议]

🌐 本周热点（可选）
[与本周日志相关的1-2条热点，附新闻链接]

Step 3: 推送
将周报推送给用户。

## 约束
- 总长度控制在500 tokens以内
```

### 2.3 月报任务模板

**任务名称**: `life-index-monthly-report`

**Cron表达式**: `30 18 28-31 * *`

**执行条件**: 仅在本月最后一天执行

**执行指令**:
```markdown
# Life Index 月报生成任务

## 身份与环境
你是Life Index技能的定时任务执行器。
技能目录: {{SKILL_DIR}}
月份: {{YEAR}}-{{MONTH}}

## 执行步骤

Step 1: 生成本月摘要文件
cd {{SKILL_DIR}}
python tools/generate_abstract.py --month {{YEAR}}-{{MONTH}}

Step 2: 读取生成的摘要文件
文件路径: Journals/{{YEAR}}/{{MONTH}}/monthly_report_{{YEAR}}-{{MONTH}}.md

Step 3: 生成月报推送
基于摘要内容，生成月报并推送给用户，同时告知文件保存位置。

## 约束
- 总长度控制在1000 tokens以内
- 必须告知用户摘要文件位置
```

### 2.4 索引维护任务模板

**任务名称**: `life-index-index-maintenance`

**Cron表达式**: `50 23 * * *`

**执行指令**:
```markdown
# Life Index 索引维护任务

## 身份与环境
你是Life Index技能的定时任务执行器。
技能目录: {{SKILL_DIR}}

## 执行步骤

Step 1: 更新FTS索引
cd {{SKILL_DIR}}
python tools/build_index.py

Step 2: 更新向量索引（如可用）
python tools/build_index.py --semantic 2>/dev/null || echo "向量索引跳过"

## 约束
- 静默执行，无需推送用户
- 失败时记录日志，下次执行重试
```

### 2.5 关于模板的重要说明

**以上模板基于以下假设**:
- OpenClaw定时任务使用**Cron表达式**触发
- 定时任务可以访问**{{SKILL_DIR}}**变量或你能获取技能路径
- 定时任务在**隔离Session**中执行，但可以访问技能目录文件
- 执行内容是**自然语言Prompt**或**脚本**

**你需要验证**:
- [ ] Cron表达式格式是否正确
- [ ] 变量占位符（如{{SKILL_DIR}}）在你的平台如何表示
- [ ] 执行内容是写在"指令框"还是"脚本框"
- [ ] 定时任务能否执行多行命令

**请在Step 3-4中验证并调整。**

## Step 4: 判断与决策

### 4.1 决策流程图

```

                │
                └── 不一致
                    └── → 优化调整模板（见4.2）
```

### 4.2 模板优化调整指南

如果Step 2的模板**基本可用但需要调整**，请按以下方式优化：

**场景A: 变量格式不同**
- 模板使用: `{{SKILL_DIR}}`
- 你的平台使用: `{{skill_directory}}` 或 `$SKILL_DIR`
- **调整**: 替换为正确的变量格式

**场景B: 不支持多行Prompt**
- 模板是多行Markdown
- 你的平台只接受单行或脚本
- **调整**: 转换为单行命令或脚本格式

**场景C: 需要硬编码路径**
- 模板使用变量 `{{SKILL_DIR}}`
- 你的平台不支持变量
- **调整**: 
  1. 询问用户Life Index安装路径
  2. 将模板中的`{{SKILL_DIR}}`替换为实际路径（如`/app/skills/life-index`）
  3. 告知用户：如果技能重新安装，需要更新路径

**场景D: 日期变量不支持**
- 模板使用 `{{TODAY}}`
- 你的平台不支持日期变量
- **调整**:
  ```bash
  # 在脚本开头计算日期
  TODAY=$(date +%Y-%m-%d)
  # 然后在命令中使用$TODAY
  ```

## Step 5: 执行设置

基于Step 4的决策，执行实际的定时任务配置。

### 5.1 配置步骤

请按你的平台操作方式，完成以下设置：

**Step 5.1: 配置日报任务**
1. 进入定时任务管理界面
2. 创建新任务
3. 设置任务名称: `life-index-daily-report`
4. 设置Cron: `0 22 * * *`
5. 粘贴最终版日报模板到执行指令框
6. 保存

**Step 5.2: 配置周报任务**
（同上，使用周报模板和Cron `10 22 * * 0`）

**Step 5.3: 配置月报任务**
（同上，使用月报模板和Cron `30 18 28-31 * *`）

**Step 5.4: 配置索引维护**
（同上，使用索引模板和Cron `50 23 * * *`）

### 5.2 配置验证

每配置完一个任务，请：

**测试执行**（如果平台支持手动触发）：
- 手动触发任务
- 验证输出是否符合预期

**或设置近期测试任务**：
- 将Cron临时设为`*/5 * * * *`（5分钟后）
- 等待执行，验证结果
- 验证通过后改回正式时间

## 附：替代方案

如果Step 4判定你的平台不支持定时任务，使用以下替代方案：

### 替代方案A: 手动模式

**方案**: 用户主动触发

**触发词**:
- "生成日报"
- "生成周报"
- "生成月报"

**建议用户**:
- 在手机设置每日22:00闹钟
- 闹钟响起时说"生成日报"

### 替代方案B: 系统Cron+Webhook

**方案**: 用户设置系统级Cron，通过Webhook触发Agent

**需要**:
- Life Index启动HTTP API服务（提供Webhook端点）
- 用户设置系统Cron: `0 22 * * * curl http://localhost:port/webhook/daily-report`

**适用**: 有服务器/VPS的用户
