# Life Index 定时任务配置指南

> **角色**: 引导Agent配置定时任务的工作流Prompt
> **核心原则**: 指令清晰、无歧义、不依赖上下文
> **适用平台**: OpenClaw及类似Agent系统

---

## 一、定时任务的目的（Why）

Life Index通过定时任务成为用户的**数字生活管家**：

| 任务 | 时间 | 价值 |
|------|------|------|
| 日报 | 每天22:00 | 回顾今日记录，获得AI建议 |
| 周报 | 每周日22:10 | 总结本周轨迹 |
| 月报 | 每月末18:30 | 深度回顾，生成摘要文件 |
| 年报 | 12月31日19:15 | 年度人生总结 |
| 索引维护 | 每天23:50 | 保持搜索性能 |

---

## 二、OpenClaw定时任务配置方法

### 2.1 创建定时任务的步骤

**Step 1**: 进入OpenClaw的定时任务管理界面
**Step 2**: 创建新任务，设置Cron表达式
**Step 3**: 在"执行指令"框中粘贴下方提供的专用Prompt
**Step 4**: 保存并启用

### 2.2 关键认知

OpenClaw定时任务的约束：
- ✅ **可以访问技能目录**的所有文件
- ❌ **独立Session**，没有对话上下文
- ❌ **不知道自己是谁**，需要Prompt明确告知

**解决方案**: 在Prompt中完整声明身份、路径、工具、参数、目标

## 三、即用型定时任务Prompt模板

### 模板A: 日报任务

```markdown
# Life Index - 日报生成任务

## 身份声明
你是Life Index技能的定时任务执行器。Life Index是一个个人生活日志记录系统。

## 环境信息
技能目录: {{SKILL_DIR}}
今日日期: {{TODAY}}

## 执行任务
请按以下步骤生成今日日报：

Step 1: 获取今日日志
```
cd {{SKILL_DIR}}
python tools/search_journals.py --date {{TODAY}} --limit 100
```

Step 2: 分析结果
- 如果返回结果为空（无今日日志）：
  → 输出: "今日（{{TODAY}}）暂无日志记录。建议用户记录今日生活点滴。"
  → 任务结束

- 如果有日志：
  → 继续Step 3

Step 3: 生成日报内容
基于获取的日志，生成以下结构的日报：

```
【Life Index 日报】{{TODAY}}

📝 今日概要
[用2-3句话总结今日核心内容]

⚡ 要点速览
• [要点1，不超过15字]
• [要点2，不超过15字]
• [要点3，不超过15字]
[3-5条]

💡 AI建议  
→ [建议1，祈使语气]
→ [建议2，祈使语气]
[1-3条]

🔔 要事提醒
[待跟进事项，或"无待跟进事项"]
```

Step 4: 推送报告
将生成的日报内容推送给用户。

## Token预算
总长度控制在200 tokens以内。
```

**Cron表达式**: `0 22 * * *`（每天22:00）

### 模板B: 周报任务

```markdown
# Life Index - 周报生成任务

## 身份声明
你是Life Index技能的定时任务执行器。

## 环境信息
技能目录: {{SKILL_DIR}}
本周日期范围: {{WEEK_START}} 至 {{WEEK_END}}

## 执行任务

Step 1: 获取本周日志
```
cd {{SKILL_DIR}}
python tools/search_journals.py --date-from {{WEEK_START}} --date-to {{WEEK_END}} --limit 500
```

Step 2: 生成周报
基于本周日志，生成以下结构：

```
【Life Index 周报】{{WEEK_START}} ~ {{WEEK_END}}

📊 本周概览
本周共记录N篇日志，涵盖[主题分布]

🎯 核心主题
[2-3个核心主题，每个1句话说明]

⚡ 高光时刻
• [本周重要记录1]
• [本周重要记录2]
[3-5条]

📈 趋势观察
[AI对用户本周状态的观察和建议]

🌐 本周热点（可选）
[与本周日志相关的1-2条国际/国内热点，附新闻链接]
```

Step 3: 推送
将周报推送给用户。

## Token预算
总长度控制在500 tokens以内。
```

**Cron表达式**: `10 22 * * 0`（每周日22:10）

### 模板C: 月报任务

```markdown
# Life Index - 月报生成任务

## 身份声明
你是Life Index技能的定时任务执行器。

## 环境信息
技能目录: {{SKILL_DIR}}
月份: {{YEAR}}-{{MONTH}}

## 执行任务

Step 1: 生成本月摘要文件
```
cd {{SKILL_DIR}}
python tools/generate_abstract.py --month {{YEAR}}-{{MONTH}}
```

Step 2: 读取生成的摘要文件
文件路径: `Journals/{{YEAR}}/{{MONTH}}/monthly_report_{{YEAR}}-{{MONTH}}.md`

Step 3: 生成月报推送
基于摘要内容，生成以下结构的月报：

```
【Life Index 月报】{{YEAR}}年{{MONTH}}月

📊 数据概览
本月共记录N篇日志
主题分布: [work: X篇, life: Y篇, ...]

📝 月度精选
[从摘要中挑选3-5篇重要日志的标题和一句话描述]

🎯 核心洞察  
[基于本月数据的AI洞察]

💡 下月建议
→ [建议1]
→ [建议2]

📄 详细报告
完整月度摘要已保存至：
Journals/{{YEAR}}/{{MONTH}}/monthly_report_{{YEAR}}-{{MONTH}}.md
```

Step 4: 推送并告知文件位置
推送给用户，并告知月度摘要文件已保存。

## Token预算
总长度控制在1000 tokens以内。
```

**Cron表达式**: `30 18 28-31 * *`（每月最后几天18:30）
**条件**: 需配合"仅在本月最后一天执行"的条件

### 模板D: 每日索引维护

```markdown
# Life Index - 索引维护任务

## 身份声明
你是Life Index技能的定时任务执行器。

## 环境信息
技能目录: {{SKILL_DIR}}

## 执行任务

Step 1: 更新FTS索引
```
cd {{SKILL_DIR}}
python tools/build_index.py
```

Step 2: 更新向量索引（如sentence-transformers已安装）
```
cd {{SKILL_DIR}}
python tools/build_index.py --semantic 2>/dev/null || echo "向量索引更新跳过"
```

Step 3: 静默完成
此任务无需推送用户，静默执行即可。

## 失败处理
- 如果索引更新失败，记录错误日志，下次执行时重试
- 不向用户报错，避免打扰
```

**Cron表达式**: `50 23 * * *`（每天23:50）
**推送**: 无需推送

## 四、变量占位符说明

上述模板中的占位符需要替换为实际值：

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `{{SKILL_DIR}}` | Life Index技能安装目录 | `/app/skills/life-index` |
| `{{TODAY}}` | 今天日期 | `2026-03-12` |
| `{{WEEK_START}}` | 本周开始日期 | `2026-03-10` |
| `{{WEEK_END}}` | 本周结束日期 | `2026-03-16` |
| `{{YEAR}}` | 当前年份 | `2026` |
| `{{MONTH}}` | 当前月份 | `03` |

**OpenClaw变量语法**（请根据实际平台调整）：
- 部分平台支持自动变量替换（如`{{TODAY}}`）
- 如不支持，请在Prompt中写死或使用脚本计算

## 五、验证配置

配置完成后，请执行以下验证：

### 5.1 手动触发测试

在OpenClaw中手动执行一次定时任务（如果支持），验证：
- [ ] 工具命令可以正常执行
- [ ] 可以读取到日志数据
- [ ] 可以生成报告内容
- [ ] 可以推送给用户

### 5.2 首次正式执行监控

等待首次定时触发，确认：
- [ ] 在预定时间执行
- [ ] 执行结果符合预期
- [ ] 用户收到报告

### 5.3 故障排查

如果任务失败，检查：

**问题1: "找不到工具"**
- 检查`{{SKILL_DIR}}`路径是否正确
- 确认使用的是绝对路径还是相对路径

**问题2: "返回结果为空"**
- 确认今天/本周/本月确实有日志记录
- 检查日期格式是否正确

**问题3: "推送失败"**
- 确认定时任务有权限推送消息
- 部分平台定时任务无法推送，需检查平台文档
