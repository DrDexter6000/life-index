# 日报输出模板

> **Token 预算**：~200 tokens
> **输出形式**：推送（不保存文件）

---

## 结构定义

| Section | Token | 内容要求 | 格式规范 |
|---------|-------|---------|---------|
| **标题** | ~10 | 【Life Index 日报】{日期} | 固定格式 |
| **今日概要** | 30-50 | 2-3 句话总结今日核心内容 | 纯文本，无 bullet |
| **要点速览** | 60-100 | 3-5 个关键事项，每个 ≤15 字 | `• ` 开头，每行一条 |
| **AI 建议** | 30-50 | 1-3 条个性化建议 | `→ ` 开头，祈使语气 |
| **要事提醒** | 0-30 | 待跟进事项（可为空） | `🔔 ` 开头，可选 |

---

## 输出示例

```
【Life Index 日报】2026-03-11

📝 今日概要
今日专注 Life Index SCHEDULE.md 优化研究，深入分析了 5 个高人气 Agent Skills 的定时任务设计模式，识别出 8 个主要问题并提出了优化建议。

⚡ 要点速览
• 完成 claude-mem、ReMe、obsidian-pkm 等项目分析
• 识别出触发词不明确、步骤不原子化等问题
• 提出了 STOP 点设计、Token 预算制等优化方案
• 确定了三阶段结构（触发→执行→输出）
• 创建了日报/周报/月报的标准化模板

💡 AI 建议
→ 建议优先实施 STOP 点设计，减少 Agent 误操作风险
→ Token 预算制可有效控制报告长度，推荐立即采用

🔔 要事提醒
无待跟进事项
```

---

## 验证清单

```
⛔ STOP. 推送前验证：

- [ ] 今日概要：2-3 句话（30-50 tokens）✓
- [ ] 要点速览：3-5 条（60-100 tokens）✓
- [ ] AI 建议：1-3 条（30-50 tokens）✓
- [ ] 要事提醒：如有待办则列出，否则"无待跟进事项" ✓
- [ ] 无敏感信息（密码、密钥等）✓
```
