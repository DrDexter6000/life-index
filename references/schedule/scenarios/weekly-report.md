# 周报场景指南

> **场景**：生成本周日志的智能周报并推送给用户
> **触发**：每周日 22:10 / 用户说"生成周报"
> **模板**：[weekly-template.md](../templates/weekly-template.md)

---

## 触发条件

| 类型 | 条件 | 行为 |
|------|------|------|
| **Immediate** | 系统时间到达周日 22:10 ± 15 分钟 + 本周有日志 | 自动执行 |
| **Manual** | 用户说"生成周报"、"weekly report"、"本周总结" | 执行，可选确认 |

---

## 执行流程

### Step 1: 获取本周日志

**命令**：
```bash
python tools/search_journals.py --date-from {MONDAY} --date-to {NOW} --limit 500
```

**参数说明**：
- `{MONDAY}`: 本周一日期，格式 `YYYY-MM-DD`
- `{NOW}`: 当前日期时间
- 示例: `python tools/search_journals.py --date-from 2026-03-09 --date-to 2026-03-15 --limit 500`

**条件判断**：
```
IF data.length == 0:
  → Skip, log "No journals this week"
  → 结束执行

IF data.length > 0:
  → Proceed to Step 2
```

---

### Step 2: 生成报告内容

**阅读模板**：[weekly-template.md](../templates/weekly-template.md)

**Token 预算**：~500 tokens

**内容结构**：

| Section | Token | 内容要求 | 格式 |
|---------|-------|---------|------|
| 标题 | ~10 | 【Life Index 周报】{周次} | 固定 |
| 本周概要 | 50-80 | 3-4 句话 | 纯文本 |
| 要点速览 | 100-150 | 5-7 条 | `• ` 开头 |
| AI 建议 | 50-80 | 2-4 条 | `→ ` 开头 |
| 要事提醒 | 0-50 | 待办 | `🔔 ` 开头 |
| 本周热点 | 100-150 | Top 5 | 编号列表 |
| 目标进度 | 30-50 | 进度更新 | 表格/文字 |

---

### Step 3: 获取国际热点

**执行方式**：
1. 使用 Agent 网络搜索能力
2. 搜索关键词：`this week world news`、`本周国际热点`
3. 筛选 Top 5，每条 1 句话描述

**超时设置**：30 秒

**失败处理**：
```
IF 热点获取失败:
  → 省略热点部分
  → 在报告中备注："热点获取失败，已跳过"
  → 继续执行 Step 4
```

---

### Step 4: 更新年度摘要

**命令**：
```bash
python tools/generate_abstract.py --year {CURRENT_YEAR}
```

**说明**：
- 静默执行
- 失败不影响周报推送
- 仅记录错误日志

---

### Step 5: 推送前验证

**⛔ STOP. 推送前验证**：

```
验证清单：
- [ ] 本周概要：3-4 句话 ✓
- [ ] 要点速览：5-7 条 ✓
- [ ] AI 建议：2-4 条 ✓
- [ ] 要事提醒：如有待办则列出 ✓
- [ ] 本周热点：Top 5（如获取成功）✓
- [ ] 目标进度：如有更新则列出 ✓

检查敏感信息：
- [ ] 无密码、密钥、隐私内容 ✓
```

---

### Step 6: 推送消息

**推送渠道**：主会话（始终） + IM渠道（已配置则推送）

**推送格式**：详见模板

---

## 错误处理

| 场景 | 处理 |
|------|------|
| 无日志 | 跳过执行 |
| 热点获取失败 | 省略热点部分，继续执行 |
| 年度摘要失败 | 记录错误，不影响周报 |
