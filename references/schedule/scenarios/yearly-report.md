# 年报场景指南

> **场景**：生成本年度日志的智能年报，保存文件并推送给用户
> **触发**：12月31日 19:15 / 用户说"生成年报"
> **Token 预算**：~3000 tokens
> **输出形式**：文件 + 推送

---

## 触发条件

| 类型 | 条件 | 行为 |
|------|------|------|
| **Immediate** | 系统时间到达 12月31日 19:15 + 本年有日志 | 自动执行 |
| **Manual** | 用户说"生成年报"、"yearly report"、"年度总结" | 执行，可选确认 |

---

## 执行流程

### Step 1: 获取本年度日志

**命令**：
```bash
python -m tools.search_journals --date-from {YEAR}-01-01 --date-to {YEAR}-12-31 --limit 10000
```

**参数说明**：
- `{YEAR}`: 年份，格式 `YYYY`
- 示例: `python -m tools.search_journals --date-from 2026-01-01 --date-to 2026-12-31 --limit 10000`

**条件判断**：
```
IF data.length == 0:
  → Skip, log "No journals this year"
  → 结束执行

IF data.length > 0:
  → Proceed to Step 2
```

---

### Step 2: 生成报告内容

**Token 预算**：~3000 tokens

**内容结构**：

| Section | Token | 内容要求 | 格式规范 |
|---------|-------|---------|---------|
| **标题** | ~10 | # Life Index 年度报告 | Markdown H1 |
| **元数据** | ~30 | 周期、时间、统计摘要 | 引用块 |
| **年度概要** | 200-300 | 3-5 段话总结全年核心内容 | 纯文本 |
| **季度回顾** | 300-400 | 4 个季度的核心事件 | 表格 + 简述 |
| **要点回顾** | 300-400 | 10-15 个年度关键事项 | 编号列表 |
| **主题分析** | 200-300 | 按主题维度分析 | 表格 |
| **项目追踪** | 150-200 | 年度项目进度 | 表格 |
| **AI 甄选** | 300-400 | Top 10 日志推荐 | 表格 + 理由 |
| **要事提醒** | 0-100 | 跨年度待办事项 | 编号列表 |
| **年度热点** | 300-400 | Top 10 国际热点 | 编号列表 |
| **下年展望** | 100-150 | 下一年度计划 | 纯文本 |

---

### Step 3: 特殊分析

#### 3.1 季度分析

**执行方式**：
1. 按季度划分日志（Q1: 1-3月, Q2: 4-6月, Q3: 7-9月, Q4: 10-12月）
2. 统计每个季度的日志数量和主要主题
3. 识别每个季度的核心事件（最多 3 个）
4. 生成季度对比表格

**输出格式**：
```
| 季度 | 核心事件 | 日志数 | 主要主题 |
|------|---------|-------|---------|
| Q1 | {事件} | {数量} | {主题} |
| Q2 | {事件} | {数量} | {主题} |
| Q3 | {事件} | {数量} | {主题} |
| Q4 | {事件} | {数量} | {主题} |
```

#### 3.2 主题分析

**执行方式**：
1. 统计每个 topic 的日志数量
2. 计算占比（日志数 / 总日志数）
3. 识别年度亮点（每个主题最重要的 1-2 个事件）
4. 生成主题洞察（3-5 句话总结）

**Topic 分类**：

| Topic | 含义 | 年度典型事件 |
|-------|------|-------------|
| work | 工作/职业 | 项目开发、职业发展 |
| learn | 学习/成长 | 技能学习、读书笔记 |
| health | 健康/身体 | 运动、医疗、情绪 |
| relation | 关系/社交 | 亲子、朋友、社交 |
| think | 思考/反思 | 人生感悟、决策复盘 |
| create | 创作/产出 | 作品发布、创意实现 |
| life | 生活/日常 | 日常记录、旅行、美食 |

#### 3.3 项目追踪

**执行方式**：
1. 统计所有涉及的项目（从 frontmatter 的 project 字段提取）
2. 追踪每个项目的年度变化
3. 标记项目状态

**状态标识**：
- ✅ 完成：项目已结束
- 🔄 进行中：项目仍在推进
- ⏸️ 暂停：项目暂时搁置
- ❌ 取消：项目已放弃

#### 3.4 AI 甄选

**执行方式**：
1. 从年度日志中筛选最重要的 10 篇
2. 推荐理由：项目里程碑、情感价值、学习收获、人生节点等
3. 格式：表格（排名、标题、日期、推荐理由）

---

### Step 4: 获取年度热点

**执行方式**：
1. 使用 Agent 网络搜索能力
2. 搜索关键词：`2026 year in review`、`年度热点`
3. 筛选 Top 10，每条 1 句话描述
4. **每条新闻后附带来源链接**（格式：`[新闻摘要](URL)`）

**超时设置**：120 秒

**失败处理**：省略热点部分，继续执行

---

### Step 5: 保存文件

**保存位置**：
```
~/Documents/Life-Index/Journals/{YEAR}/report_{YEAR}.md
```

**示例**：
```
~/Documents/Life-Index/Journals/2026/report_2026.md
```

**⛔ STOP. 保存前验证**：
```
验证清单：
- [ ] 文件路径正确：{YEAR}/ ✓
- [ ] 文件名格式：report_{YEAR}.md ✓
- [ ] 内容格式：Markdown，包含所有必填部分 ✓
- [ ] Token 预算：~3000 tokens ✓
```

---

### Step 6: 推送前验证

**⛔ STOP. 推送前验证**：

```
结构验证：
- [ ] 标题格式正确 ✓
- [ ] 元数据完整：周期、时间、统计摘要 ✓
- [ ] 年度概要：3-5 段话 ✓
- [ ] 季度回顾：4 个季度 ✓
- [ ] 要点回顾：10-15 条 ✓
- [ ] 主题分析：所有主要主题 ✓
- [ ] 项目追踪：至少 1 个项目 ✓
- [ ] AI 甄选：Top 10 日志 ✓
- [ ] 年度热点：Top 10 ✓
- [ ] 下年展望：1-5 条 ✓

内容验证：
- [ ] 数据准确：日志数量、活跃天数等 ✓
- [ ] 无敏感信息 ✓
- [ ] 格式统一 ✓
```

---

### Step 7: 推送消息

**推送渠道**：主会话 + 已配置渠道

**消息内容**：
- 报告概要
- 文件路径
- 统计摘要
