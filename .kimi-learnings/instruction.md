# Kimi Learning Workflow — 固定操作要求

> 每轮交流结束后自动阅读本文档并按要求完成操作。

## 1. 目录结构

```
.kimi-learnings/
├── instruction.md          # 本文档（操作规范 SSOT）
├── kimi-learnings.md       # 跨轮次抽象洞察总结（增量维护）
└── round-{N}/              # 每轮一个目录
    └── exchanges.md        # 本轮交流概述
```

## 2. 每轮结束后的强制动作（按顺序）

### Step 1: 撰写 exchanges.md
在 `round-{N}/exchanges.md` 中记录：
- **任务接收**：Kimi 接到了什么任务/指令
- **执行动作**：执行了哪些工具调用、文件修改、代码变更
- **遇见问题**：遇到了什么障碍、错误、意外
- **采取动作**：如何诊断、如何修复、决策依据
- **最终交付**：交付了什么成果、指标、baseline
- **Opus 4.7 评价**：关键评价、拍板、驳回、条件接受的具体内容

### Step 2: 增量更新 kimi-learnings.md
在文档末尾追加本轮的抽象洞察：
- **Kimi 的思维特征**：本轮 Kimi 倾向于什么模式（如过早标记✅、低估 regression、过度复杂化）
- **Opus 4.7 的思维特征**：Opus 4.7 的核心判断逻辑（如严格守底线、要求数据先于结论、强制诊断步骤）
- **关键差异**：本轮两者在哪些点上分歧最大，最终结果证明哪方正确
- **可复用的教训**：下一轮应该直接采用的做法，避免重复踩坑

### Step 3: 阅读 instruction.md
确保下一轮开始时重新阅读本文档，唤醒流程记忆。

## 3. 质量标准

- exchanges.md 必须包含 Opus 4.7 的原话评价（或高度准确的概括）
- kimi-learnings.md 不得流于表面，必须提炼到"思维模式"层面
- 不得跳过任何一轮，即使该轮只有简单问答
