## 🎉 写在最前

这是我在 GitHub 创建的第一个仓库。

我创建它不是为了展示代码能力，而是为了建立一个可以陪伴我几十年的**人生档案馆**。我想记录下那些真正重要的东西——比如女儿团团两岁时的样子，那个被称为"尿片侠"的小肉坨坨，如何在阳台上掏出烟盒模仿我的动作。

我想在三年后、十年后，还能准确地回答一个问题：**"那时的我，经历了什么？"**

---

## 📖 Life Index 是什么？

**Life Index** 是一个 **Agent-first、本地优先** 的个人生活记录系统。

在这个 AI 时代，我们选择了一条反直觉的路：**不要云端同步，不要复杂功能，不要自动化陷阱**。我们要的很简单：

> **用自然语言记录生活，让 Agent 帮你整理归档，最终以清晰的索引呈现你的人生轨迹。**

```yaml
# 想念尿片侠

在翻过往资料的时候，看到了团团小时候的照片，那个只有2岁上下的尿片侠

突然有一种伤感 —— 我好想这个小娃娃，好想再见她一面，好想再能体验一次把小肉坨坨抱在怀里的感觉...

三岁以前的娃娃好像小动物，全凭直觉驱动...我还记得，她屁颠屁颠儿跑到阳台上去，打开我的烟盒子、然后掏出一根烟递给我 —— 虽然她不知道这是在干嘛，但那是观察爸爸生活习惯的总结[笑哭]

三岁开始，团团上幼儿园了...那个让我神魂颠倒的尿片侠、她长大了

总而言之，小疙瘩，爸爸想你了
```

这就是 Life Index 的使命：**保存那些会让你在未来某个深夜突然鼻酸的细节。**

## 🏛️ 核心架构

### 1. Agent-first
不用学习复杂的标签系统或软件操作。直接告诉 Agent：
> "记录一下，今天看到团团以前的照片，突然很想念那个2岁的尿片侠"

Agent 会自动提取元数据（时间、地点、人物、情绪、标签），生成结构化的 Markdown 文件。

### 2. 本地优先 / 数据主权
- **100% 本地存储**：所有数据以 Markdown + YAML 格式保存在你的硬盘
- **人可读**：即使 20 年后没有 Life Index，你依然可以直接阅读纯文本文件
- **绝对隐私**：没有任何云服务，你的"尿片侠"记忆只属于你自己

### 3. 三层索引模型
我们不搞复杂的文件夹嵌套，只用三层清晰的维度：

| 层级 | 维度 | 示例 |
|:---:|:---:|:---:|
| **L1 Topic** | 主题（知识维度）| `think`（思考）、`create`（创作）、`relation`（关系） |
| **L2 Project** | 项目（目标维度）| `LifeIndex`（本项目）、`Parenting`（育儿） |
| **L3 Tag** | 标签（特征维度）| `亲子`、`回忆`、`感伤`、`尿片侠` |

你想找"所有关于团团的感伤回忆"？搜索 `团团` + `感伤` 即可，无论是在哪个项目下。

## 🚀 快速开始

### 前置要求
- 一个支持文件操作的 AI Agent（如本项目的配套 Agent）
- 一个本地目录（建议：`~/Documents/Life-Index`）

### 目录结构
```
Life-Index/
├── journals/          # 日志主目录（按年月组织）
│   └── 2026/
│       └── 03/
│           └── life-index_2026-03-04_002.md
├── attachments/       # 附件（照片、视频、语音）
│   └── 2026/03/
│       └── [视频文件].mp4
└── by-topic/          # 自动生成的索引
    ├── 主题_think.md
    ├── 项目_LifeIndex.md
    └── 标签_亲子.md
```

### 记录你的第一篇日志

直接对 Agent 说：

> "2026年3月5日，北京，晴天。今天想记录一下关于 Life Index 的初心。这个项目不是为了做第二个 Notion 或 Obsidian，而是为了帮助像我这样的普通人整理人生碎片。我想记录下女儿成长的细节，比如她两岁时模仿我抽烟的样子..."

Agent 会：
1. 生成带 frontmatter 的 Markdown 文件
2. 自动提取时间、地点、天气、标签
3. 更新月度摘要和主题索引
4. 保持纯文本，永不过时

## 📚 文档导航

Life Index 遵循 **SSOT（单一事实来源）** 原则：

| 文档 | 内容 | 适用场景 |
|:---:|:---:|:---:|
| **[HANDBOOK.md](./HANDBOOK.md)** | 项目愿景、架构设计、核心原则 | 你想理解"为什么这样设计" |
| **[AGENT.md](./AGENT.md)** | Agent 执行指令、工具接口、工作流 | 你是开发者，想实现 Agent 功能 |
| **[CHANGELOG.md](./CHANGELOG.md)** | 重大决策记录、版本演进 | 你想了解项目历史 |

## 🎯 设计底线

```
宁可功能简单，不可系统复杂
宁可人工维护，不可自动化陷阱  
宁可牺牲性能，不可牺牲可靠性
```

我们不做：
- ❌ 云端同步（你自己可以用云盘备份，但我们不强制）
- ❌ 富文本编辑（Markdown 足够表达"想念尿片侠"的情感）
- ❌ 实时协作（这是你的人生，不是团队的）
- ❌ AI 自动总结（我们索引，但不替你思考）

## 🌱 适合谁？

- **新手父母**：想记录下孩子每一个"尿片侠"时刻，不想被复杂的育儿 App 绑架
- **数字游民**：在不同城市间穿梭，需要记录地理位置和当地心境
- **长期主义者**：想要一个能用 30 年的系统，而不是追逐最新的生产力工具
- **隐私偏执狂**：认为自己的生活细节不该成为云服务商的训练数据
