## 🎉 写在最前

这是我在 GitHub 创建的第一个仓库。  

我创建它——  
不是为了展示编程技术，而是因为我自己迫切需要一个专门**整理和存放人生碎片的工具**。  
不是专门为知识管理打造，也不是为了效率提升，而是为了建立一个**属于我自己的人生档案馆**。

**【记录引擎】**  
把零散的灵感、阅读批注、对话碎片、瞬间的情绪波动转化为结构化数据。不是写给别人看的博客，而是带时间戳、地理位置、情绪标签、关联人物的数字化切片。让那个"上周突然想到但忘了具体是什么"的念头，或者"三年前某个下午空气中的味道"，都能被完整捕获并永久存储在本地。

**【索引引擎】**  
通过多维度标签（时间/主题/人物/情绪）在任意时刻快速定位过往。验证当年的直觉是否正确，复盘创业失败的具体决策节点，或者回答那个偶尔会冒出来的问题："十年前的我，为什么做出那个决定命运的决定？"

**但以上只是技术说明。**    
> *对我来说，Life Index 真正的用法发生在深夜：当我偶然翻到女儿两岁时的照片，那种幸福中带着怅然若失的复杂情绪，可以被准确地固定下来——不仅是她的笑脸，更是我当时作为父亲的心跳、阳台上的光线、以及那个知道"这一刻正在消逝"的清醒。*
> > *这些记录最终会成为一本数字化家书。也许有一天我不在人世，她打开这些文件，就像翻开一本泛黄的老书，读到的不仅是爸爸的爱，还有爸爸这一生跌撞出来的经验、犯过的错误与得来的智慧。*

**🎯 **我相信： 每个人都值得拥有这样一份人生档案馆。**  

> —— 凭什么只有政治家/有钱人才能聘请作家撰写回忆录？
> —— 你还记得十年前的那个决定，是基于什么做出的吗？
> —— 你想不想留给下一代一本基于真实人生写就的经验宝典、错题集，甚至是数字化的记忆遗产？  

**Life Index，就是凡人也能拥有的人生数据资产。**

## 📖 Life Index 是什么？

**Life Index** 是一个 **Agent-first**、**Offline-must（离线优先）** 的个人生活记录工具。

在这个 AI 和云端一切的时代，我选择了一条反直觉的路：**不要云端同步，不要功能膨胀，不要自动化陷阱**。我要的很简单：

> **用自然语言讲述你的生活、你的想法、你的感受、你读过的书、你分享的观点。让 Agent 帮你整理归档，最终以清晰的索引呈现你的人生轨迹。**

```yaml
# 想念小英雄

在翻过往资料的时候，看到了乐乐小时候的照片，那个只有2岁上下的小英雄

突然有一种伤感 —— 我好想这个小娃娃，好想再见她一面，好想再能体验一次把小肉坨坨抱在怀里的感觉

三岁以前的娃娃好像小动物，全凭直觉驱动，想吃饭、想睡觉、想玩耍、想要爸爸要妈妈 —— 都不会经过大脑思考。

而且这个世界对于她来说好危险，什么都不懂、什么都不知道，最好奇、最感兴趣的就是爸爸妈妈的一举一动，天天围着爸爸妈妈转、模仿爸爸妈妈的一举一动。

我还记得，她屁颠屁颠儿跑到阳台上去，打开我的烟盒子、然后掏出一根烟递给我 —— 虽然她不知道这是在干嘛，但那是观察爸爸生活习惯的总结[笑哭]

三岁开始，乐乐上幼儿园了，开始接触老师和同学了 —— 她变成小孩儿了，开始有自己的思维/思考，有自己的个性甚至计划，爸爸妈妈也不是她世界中唯一的重心

三岁之后的乐乐依然是全宇宙最重要最珍贵的存在 —— 但确实，她已经不是那个咿呀学语的小婴孩儿了，那个让我神魂颠倒的小英雄、她长大了

我既希望我们一家人永恒停留在乐乐2岁的时光 —— 也盼望她长大可以去感受更美好的世界

总而言之，小豆丁，爸爸想你了

乐乐，这个项目就是爸爸为了你而建的。
```
</details>

这就是 Life Index 的使命：**保存那些会让你在未来某个深夜突然鼻酸的细节。**

## 🏛️ 核心架构

### 1. Agent-first
不用学习复杂的标签系统或软件操作。直接告诉 Agent：
> "记录一下，今天看到乐乐以前的照片，突然很想念那个2岁的小英雄，那种甜蜜又忧伤的感觉"

Agent 会自动提取元数据（时间、地点、人物、情绪、标签），生成结构化的 Markdown 文件。

### 2. 本地优先 / 数据主权
- **100% 本地存储**：所有数据以 Markdown + YAML 格式保存在你的硬盘
- **人可读**：即使 20 年后没有 Life Index，你依然可以直接阅读纯文本文件
- **绝对隐私**：没有任何云服务，你的"小英雄"记忆只属于你自己

### 3. 三层索引模型
我们不搞复杂的文件夹嵌套，只用三层清晰的维度：

| 层级 | 维度 | 示例 |
|:---:|:---:|:---:|
| **L1 Topic** | 主题（知识维度）| `think`（思考）、`create`（创作）、`relation`（关系） |
| **L2 Project** | 项目（目标维度）| `LifeIndex`（本项目）、`Parenting`（育儿） |
| **L3 Tag** | 标签（特征维度）| `亲子`、`回忆`、`感伤`、`小英雄` |

你想找"所有关于乐乐的感伤回忆"？搜索 `乐乐` + `感伤` 即可，无论是在哪个项目下。

## 🚀 快速开始

### 前置要求
- 一个支持文件操作的 AI Agent（如本项目的配套 Agent）
- 一个本地目录（建议：`~/Documents/Life-Index`）

### 目录结构
```
Life-Index/
├── journals/          # 日志主目录（按年月组织）
│   └── 2026/
│       └── 03/
│           └── life-index_2026-03-04_002.md
├── attachments/       # 附件（照片、视频、语音）
│   └── 2026/03/
│       └── [视频文件].mp4
└── by-topic/          # 自动生成的索引
    ├── 主题_think.md
    ├── 项目_LifeIndex.md
    └── 标签_亲子.md
```

### 记录你的第一篇日志

直接对 Agent 说：

> "2026年3月5日，重庆，晴天。今天想记录一下关于 Life Index 的初心。这个项目不是为了做第二个 Notion 或 Obsidian，而是为了帮助像我这样的普通人整理人生碎片。我想记录下女儿成长的细节，以及作为父亲的我的感受，比如她两岁时模仿我抽烟的样子..."

Agent 会：
1. 生成带 frontmatter 的 Markdown 文件
2. 自动提取时间、地点、天气、标签
3. 更新月度摘要和主题索引
4. 保持纯文本，永不过时

## 📚 文档导航

Life Index 遵循 **SSOT（单一事实来源）** 原则：

| 文档 | 内容 | 适用场景 |
|:---:|:---:|:---:|
| **[HANDBOOK.md](./HANDBOOK.md)** | 项目愿景、架构设计、核心原则 | 你想理解"为什么这样设计" |
| **[AGENT.md](./AGENT.md)** | Agent 执行指令、工具接口、工作流 | 你是开发者，想实现 Agent 功能 |
| **[CHANGELOG.md](./CHANGELOG.md)** | 重大决策记录、版本演进 | 你想了解项目历史 |

## 🎯 设计底线

```
宁可功能简单，不可系统复杂
宁可人工维护，不可自动化陷阱  
宁可牺牲性能，不可牺牲可靠性
```

我们不做：
- ❌ 云端同步（你自己可以用云盘备份，但我们不强制）
- ❌ 富文本编辑（Markdown 足够表达"想念小英雄"的情感）
- ❌ 实时协作（这是你的人生，不是团队的）
- ❌ AI 自动总结（我们索引，但不替你思考）

## 🌱 适合谁？

- **新手父母**：想记录下自己作为父母的真实体验——那些疲惫、惊奇、心碎的时刻，而不仅是孩子的里程碑
- **数字游民**：在不同城市间穿梭，需要记录地理位置和当地心境
- **长期主义者**：想要一个能用 30 年的系统，而不是追逐最新的生产力工具
- **隐私偏执狂**：认为自己的生活细节不该成为云服务商的训练数据
- **每个凡人**：相信自己的人生碎片值得被保存，即使这些碎片并不"高效"或"有用"
