## ⚠️ 关键区分：Life Index 是什么，不是什么

当前个人数据管理领域有三类主流方案，但 Life Index 与它们有本质不同：

| 维度 | **Life Index**<br>（人生档案馆） | **传统知识管理（PKM）**<br>（如 Notion/Obsidian） | **Agent 系统记忆优化**<br>（如 Mem0/Zep） |
|:---:|:---:|:---:|:---:|
| **核心逻辑** | **时间 + 主题**<br>（When + What） | **主题 + 链接**<br>（What + How it connects） | **及时 + 进化**<br>（Now + Keep updating） |
| **数据组织** | 按时间轴归档，可回溯到"三年前的今天" | 按知识图谱组织，强调概念关联 | 按上下文窗口优化，强调当前相关性 |
| **数据寿命** | **永久保留**<br>（20年后仍能读取） | 长期保留，但会随知识更新而重构 | **临时/进化性保留**<br>（旧记忆被压缩或遗忘） |
| **服务对象** | **人**（为你保留人生切片） | **人**（为你管理知识资产） | **Agent**（为 AI 提升任务表现） |
| **存储位置** | **用户设备文件夹**<br>（如 `~/Documents/Life-Index`）<br>📁 与 Agent 程序物理隔离 | 本地或云端（用户选择） | **Agent 配置目录**<br>（如 `AppData/Roaming/Agent`）<br>🤖 服务 AI 性能 |
| **典型用途** | 记录"那天阳光很好，2岁女儿唱了一首歌" | 整理"复利投资的 7 个核心概念" | 记住"用户昨天让我订了披萨，不要芝士" |

**关键洞察：**

- **PKM** 回答的是 **"我知道什么？"**（知识维度）
- **Agent 记忆** 回答的是 **"我刚才说了什么？"**（上下文维度）  
- **Life Index** 回答的是 **"那时的我，是什么心情？"**（时空维度）

**核心区别：Agent 记忆是消耗品，Life Index 是遗产**

Agent 记忆系统是为了让 AI **更好地服务你**（记住你的偏好、习惯、近期指令），数据存储在 Agent 的配置目录里，会随着与 AI 的交互不断**被覆盖、压缩、遗忘**——就像人类的短期记忆一样，旧细节会被新信息稀释。

Life Index 是为了让你**保留自己**（记录你的感受、决策、人生切片），数据存储在**你自己的文件夹**，与 Agent **物理隔离**，**永久保留**——就像写给未来的信，20年后打开，字迹依然清晰。

## 🛠️ 核心引擎

### 【记录引擎】
把零散的灵感、阅读批注、对话碎片、瞬间的情绪波动转化为**结构化数据**。不是写给别人看的日记，而是带时间戳、地理位置、情绪标签、关联人物的数字化切片。让那个"上周突然想到但忘了具体是什么"的念头，或者"三年前某个下午空气中的味道"，都能被完整捕获并永久存储在本地。

**与 PKM 的区别：** PKM 记录的是"这个知识点属于哪个主题"，Life Index 记录的是"这个念头发生在什么时间、什么心境下"。

**与 Agent 记忆的区别：** Agent 记忆为了当下对话的连贯性而记录，Life Index 为了未来的回溯而记录——即使那时候 Agent 已经被卸载重置，你的人生档案依然完好。

### 【索引引擎】
通过多维度标签（**时间/主题/人物/情绪**）+ 全文搜索（FTS5） + 语义搜索（RAG）在任意时刻快速定位过往。验证当年的直觉是否正确，复盘创业失败的具体决策节点，或者回答那个偶尔会冒出来的问题：**"十年前的我，到底在想什么？"**

**与 PKM 的区别：** PKM 的索引是为了找到"相关的知识"，Life Index 的索引是为了找到"那时的自己"。

**与 Agent 记忆的区别：** Agent 记忆的检索是为了"接上刚才的话茬"，Life Index 的检索是为了"与过去的自己重逢"。

## 💭 但以上只是技术说明

对我来说，Life Index 真正的用法发生在深夜：当我偶然翻到女儿两岁时的照片，那种**幸福中带怅然若失的复杂情绪**，可以被准确地固定下来——不仅是她的笑脸，更是我当时作为父亲的心跳、阳台上的光线、以及那个知道"这一刻正在消逝"的清醒。

这些记录最终会成为一本**数字化家书**。也许有一天我不在人世，她打开这些文件，就像翻开一本泛黄的老书，读到的不仅是爸爸的爱，还有爸爸这一生跌撞出来的经验、犯过的错误与得来的智慧。

**这不是 Agent 的"记忆"——它不会帮我订外卖或写代码。这是我留给女儿的"遗产"，是只有人类才需要的"过去"。**

## 📖 Life Index 是什么？

**Life Index** 是一个 **Agent-first**、**Offline-must（离线优先）** 的个人生活记录工具。

在这个 AI 和云端一切的时代，我选择了一条反直觉的路：**不要云端同步，不要功能膨胀，不要自动化陷阱**。我要的很简单：

> **用自然语言讲述你的生活、你的想法、你的感受、你读过的书、你分享的观点。让 Agent 帮你整理归档，最终以清晰的索引呈现你的人生轨迹。**

在翻过往资料的时候，看到了团团小时候的照片，那个只有2岁上下的尿片侠

突然有一种伤感 —— 我好想这个小娃娃，好想再见她一面，好想再能体验一次把小肉坨坨抱在怀里的感觉

而且这个世界对于她来说好危险，什么都不懂、什么都不知道，最好奇、最感兴趣的就是爸爸妈妈的一举一动，天天围着爸爸妈妈转、模仿爸爸妈妈的一举一动。

## 🏛️ 核心架构

### 1. Agent-first
不用学习复杂的标签系统或软件操作。直接告诉 Agent：
> "记录一下，今天看到团团以前的照片，突然很想念那个2岁的尿片侠，那种甜蜜又忧伤的感觉"

Agent 会**协助整理**元数据（时间、地点、人物、情绪、标签）——你可以直接自然语言描述，Agent 会从你的叙述中识别并提取关键信息，生成结构化的 Markdown 文件。

### 2. 本地优先 / 数据主权
- **100% 本地存储**：所有数据以 Markdown + YAML 格式保存在你的硬盘
- **人可读**：即使 20 年后没有 Life Index，你依然可以直接阅读纯文本文件
- **绝对隐私**：没有任何云服务，你的"尿片侠"记忆只属于你自己

### 3. 三层索引模型
我们不搞复杂的文件夹嵌套，只用三层清晰的维度：

| 层级 | 维度 | 示例 |
|:---:|:---:|:---:|
| **L1 Topic** | 主题（知识维度）| `think`（思考）、`create`（创作）、`relation`（关系） |
| **L2 Project** | 项目（目标维度）| `LifeIndex`（本项目）、`Parenting`（育儿） |
| **L3 Tag** | 标签（特征维度）| `亲子`、`回忆`、`感伤`、`尿片侠` |

你想找"所有关于团团的感伤回忆"？搜索 `团团` + `感伤` 即可，无论是在哪个项目下。

## 🚀 快速开始

### 前置要求
- 一个支持文件操作的 AI Agent（例如Openclaw, Lobster AI）
- 一个本地目录（建议：`~/Documents/Life-Index`）

### 安装依赖

**必需依赖：**
- Python 3.11+（核心工具运行环境）

**可选依赖：**
```bash
# 安装语义搜索依赖（推荐）
pip install -e .[semantic]

# 或手动安装
pip install sentence-transformers
```

> **说明**：sentence-transformers 用于语义搜索功能。如未安装，系统会自动跳过向量索引，全文搜索（FTS5）仍可正常使用。

### 自动化与定时任务

Life Index 支持通过 Agent 的定时任务功能实现自动化报告：

- **日报**：每日 22:00 生成并推送
- **周报**：每周日 22:10 生成并推送，同时更新年度摘要
- **月报**：每月最后一天 18:30 生成文件并推送
- **年报**：每年 12月31日 19:15 生成文件并推送
- **向量索引更新**：每日 23:50 增量更新，每月1日 03:30 全量重建

**详见 [SCHEDULE.md](./references/schedule/SCHEDULE.md) 了解完整的定时任务配置指南。**
### 目录结构
```
Life-Index/
├── Journals/          # 日志主目录（按年月组织）
│   └── 2026/
│       └── 03/
│           └── life-index_2026-03-04_002.md
├── attachments/       # 附件（照片、视频、语音）
│   └── 2026/03/
│       └── [视频文件].mp4
└── by-topic/          # 自动生成的索引
    ├── 主题_think.md
    ├── 项目_LifeIndex.md
    └── 标签_亲子.md
```

### 记录你的第一篇日志

直接对 Agent 说：

> "今天想记录一下关于 Life Index 的初心。这个项目不是为了做第二个 Notion 或 Obsidian，而是为了帮助像我这样的普通人整理人生碎片。我想记录下女儿成长的细节，以及作为父亲的我的感受，比如她两岁时模仿我用电脑工作的样子..."

Agent 会：
1. **自动填充**：地点（默认 "Chongqing, China"）、天气（自动查询）、时间（当前时间）
2. **语义提取**：从内容中识别主题、标签、心情、人物
3. **生成文件**：带 frontmatter 的 Markdown 文件
4. **确认询问**："日志已保存。地点：Chongqing, China；天气：Slight rain (小雨)。请确认是否正确？"
5. **更新索引**：自动更新月度摘要和主题索引

**你也可以主动提供信息**：
> "今天在北京开会，晴天。讨论了 Life Index 的架构设计..."

Agent 会使用你提供的地点和天气，跳过自动填充步骤。

## 🛠️ 开发者指南

### 命令行工具

Life Index 提供完整的命令行工具集，供开发者或高级用户直接使用：

```bash
# 写入日志
python tools/write_journal.py --data '{
  "date": "2026-03-07",
  "title": "完成搜索功能优化",
  "content": "今天优化了 Life Index 的搜索功能...",
  "topic": ["work"],
  "project": "Life-Index",
  "tags": ["重构", "优化"],
  "mood": ["专注", "充实"],
  "people": ["团团"]
}'

# 搜索日志（全文）
python tools/search_journals.py --query "重构" --level 3

# 搜索日志（语义搜索）
python tools/search_journals.py --query "学习笔记" --semantic

# 按多维度过滤搜索
python tools/search_journals.py --topic work --mood "专注,充实" --people "团团"

# 生成月度摘要
python tools/generate_abstract.py --month 2026-03

# 生成年度摘要
python tools/generate_abstract.py --year 2026

# 更新搜索索引
python tools/build_index.py
```

### 技术特性

| 特性 | 说明 |
|:---:|:---|
| **存储格式** | Markdown + YAML Frontmatter，纯文本，永不过时 |
| **全文搜索** | SQLite FTS5，支持 BM25 相关性排序 |
| **语义搜索** | 基于 Sentence Transformers 的向量嵌入，支持语义相似度检索 |
| **索引架构** | L1 索引层 → L2 元数据层 → L3 内容层，分层检索兼顾速度与精度 |
| **混合排序** | 支持 FTS 与语义搜索的加权混合排序（默认 FTS 0.6 + 语义 0.4） |
| **数据隔离** | 所有用户数据存储在 `~/Documents/Life-Index/`，与 Skill 目录物理隔离 |

### API 接口

所有工具返回标准 JSON 格式：

```json
{
  "success": true,
  "data": { ... },
  "error": "错误信息（如有）"
}
```

## 🎯 设计底线

```
宁可功能简单，不可系统复杂
宁可人工维护，不可自动化陷阱  
宁可牺牲性能，不可牺牲可靠性
```

我们不做：
- ❌ 云端同步（你自己可以用云盘备份，但我们不强制）
- ❌ 富文本编辑（Markdown 足够表达"想念尿片侠"的情感）
- ❌ 实时协作（这是你的人生，不是团队的）
- ❌ AI 自动总结（我们索引，但不替你思考）

## 🌱 适合谁？

- **新手父母**：想记录下自己作为父母的真实体验——那些疲惫、惊奇、心碎的时刻，而不仅是孩子的里程碑
- **数字游民**：在不同城市间穿梭，需要记录地理位置和当地心境
- **长期主义者**：想要一个能用 30 年的系统，而不是追逐最新的生产力工具
- **隐私偏执狂**：认为自己的生活细节不该成为云服务商的训练数据
- **每个凡人**：相信自己的人生碎片值得被保存，即使这些碎片并不"高效"或"有用"
