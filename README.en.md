## Why Life Index Exists

Everyone is building memory for AI.

Your knowledge base manages what you've learned. Your Agent remembers what you've said. The entire tech world is racing to make information more efficient and AI more capable.

**But has anyone asked what you, as a human being, are quietly forgetting?**

> Your Notion captures what you know вҖ” but not the way your heart skipped the first time you saw her.<br>
> Your Agent remembered last week's decision вҖ” until a scoring algorithm marked it *"no longer relevant."*<br>
> That decision that changed your life ten years ago вҖ” do you still remember how it felt?

| | **Life Index** | **Knowledge Base** (Notion/Obsidian) | **Agent Memory** (Mem0/Zep) |
|:---|:---|:---|:---|
| **Core question** | "What was I feeling then?" | "What do I know?" | "What did I just say?" |
| **Retention** | **Permanent вҖ” as legacy** | Long-term, restructured as thinking evolves | Ephemeral, evicted by scoring systems |
| **Ownership** | **Entirely yours** вҖ” local Markdown never expires | Yours, but format drifts with software | Owned by the provider, subject to deletion |
| **Cost of failure** | None вҖ” plain text reads forever | Export headaches, format lock-in | Gone instantly, lost without notice |

Life Index does exactly one thing: **keeps your life fragments searchable, forever.**

## Architecture Philosophy

Life Index isn't a weekend side project. Every layer of its design points to the same question: **how do you keep personal memories safe for fifty years?**

### Agent-Native, Not Agent-First

> Agent-First means *"we prioritize what Agents need."*
> Agent-Native means *"this system was born to be used by Agents."*

Life Index's CLI isn't a human command-line tool "with AI support bolted on." Its structured signal system, confirmation workflows, and enum-based error codes are native to how Agents operate вҖ” from the very first line of code. Agents don't have to parse natural-language error messages; they receive enums and deterministic recovery strategies.

But *Agent-Native* doesn't mean *"Agent-only."* It means we give Agents what they need most вҖ” **precise machine interfaces** вҖ” while giving humans what they need most: **natural conversation and visual experience.**

**Key design principle**: deterministic operations (writing entries, filtering by tag, browsing timelines) call the CLI Core directly, with latency under 50ms. Only operations that genuinely require *thinking* (semantic search, memoir generation, digital persona) route through the Intelligence Layer. **If it can be answered deterministically, don't ask the Agent. If it takes thought, that's what the Agent is for.**

### Three Design Principles

```
Prefer simplicity over complexity.
Prefer manual maintenance over automation traps.
Prefer reliability over performance.
```

What we deliberately don't do: вң• Cloud sync В· вң• Rich-text editing В· вң• Real-time collaboration В· вң• AI thinking on your behalf

## Quick Start

**Who this is for**: anyone who just wants their Agent to handle installation and setup вҖ” no code editing required.

> If your Agent platform already manages a canonical skill directory or checkout, prefer reusing it instead of creating a duplicate checkout in a generic workspace.

**Copy this to your Agent** вҖ” paste the following into your Agent (Claude Desktop, Cursor, OpenClaw, etc.):

```text
Please read and follow `AGENT_ONBOARDING.md` in this repository to complete
the full Life Index install, initialization, and verification:
https://github.com/DrDexter6000/life-index/blob/main/AGENT_ONBOARDING.md

Requirements:
1. Refresh and read the current authority files first, in this order:
   `bootstrap-manifest.json`, then every path listed in its
   `required_authority_docs`: `AGENT_ONBOARDING.md`, `SKILL.md`,
   `docs/API.md`, `docs/ARCHITECTURE.md`, `tools/lib/AGENTS.md`, `README.md`

   `.venv` before making any route decision. Don't skip sync just because
   files exist or `health` looks fine.
3. Route decision (fresh-install / upgrade / repair) must happen AFTER
   authority refresh + checkout sync.
4. All Python/CLI commands must use the virtual environment path.
5. If any step fails, stop immediately and report the exact error.
6. Report back in English per the format specified in the document.
```

вҶ’ Make sure you're using `.venv/bin/python` (not system python), and running from the skill root directory

**Fresh install shows "degraded" health**
вҶ’ If you haven't run `life-index index` yet, this is expected. Initialize the index first, then re-run health.

**Windows: `write --data '{...}'` is hard to escape**
вҶ’ Use `life-index write --data @first-entry.json` instead (auto-generated by the Agent during onboarding)

**Semantic search unavailable**
вҶ’ Run `.venv/bin/life-index health` to check whether sentence-transformers is installed

**venv corrupted (after Python upgrade or system migration)**
вҶ’ Delete the `.venv` directory, then re-run `python3 -m venv .venv && .venv/bin/pip install -e .`

**Upgrading to a new version**
вҶ’ Sync the canonical checkout first, then follow the freshness / sync / repair rules in `AGENT_ONBOARDING.md`

</details>
