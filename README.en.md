# 💭 Why It Exists

## 🎉 First Things First

This is my very first GitHub repository.

I'm just an ordinary father with zero coding background — I couldn't even finish this README without AI assistance.    
I didn't create this to show off technical skills. I built it because I desperately needed a **dedicated tool for preserving life's fragments** —

> **Not a knowledge base (PKM). Not an Agent memory cache. A personal life archive.**  
> **Not for knowledge retention. Not for AI performance. Only for retrieving my own life fragments.**

---

## 📌 Key Distinction: What Life Index Is Not

| Dimension | **Life Index** | **PKM** (Notion/Obsidian) | **Agent Memory** (Mem0/Zep) |
|:---|:---|:---|:---|
| **Core question** | "What was I feeling then?" | "What do I know?" | "What did I just say?" |
| **Retention** | **Permanent — as legacy** | Long-term, restructured as thinking evolves | Ephemeral, discarded after use |
| **Ownership** | **Entirely yours** — local files never expire | Yours, but format may drift with software updates | Owned by service provider, subject to deletion |
| **Cost of failure** | None — Markdown files remain readable forever | Export headaches, format lock-in | Gone in an instant, lost without warning |

> **Agent memory is consumable** — it helps you order food, write code, then vanishes into the void.  
> **Life Index is legacy** — it records why you cried, who made you laugh, until the end of time.

---

## 💭 But That's Just the Technical Pitch

For me, Life Index truly comes alive at midnight: when I stumble upon photos of my daughter at age two, and that **bittersweet ache — joy laced with loss** — can finally be pinned down. Not just her smile, but my heartbeat as her father that day, the light on the balcony, and the quiet awareness that *this moment is already passing*.

These records will become **digital letters to the future**. Perhaps one day when I'm no longer here, she'll open these files like a faded old book — reading not just her father's love, but also the hard-won wisdom from a lifetime of stumbles, mistakes, and lessons learned.

**This is not "memory" for an Agent — it won't help me order takeout or write code. This is a legacy for my daughter — a "past" that only humans need.**

## 🎯 What Life Index Believes

Everyone deserves a personal life archive.

—— **Must only politicians and the wealthy hire writers for their memoirs?**    
—— **Are only Elon Musk's daily moments and opinions worthy of being recorded?**   
—— **Do you still remember what drove that life-changing decision you made ten years ago?**  
—— **Wouldn't you want to leave the next generation a playbook of real-life lessons, a collection of hard-won mistakes, or even a digital memory legacy?**

<u>This is digital equity for ordinary people</u> — not about optimizing Agents, but about recording yourself.

And once you've accumulated enough of "yourself," perhaps one day these records will travel further than you ever imagined.

---

## 🌱 Who Is This For?

- **New parents**: Capturing the raw reality of parenthood — the exhaustion, the wonder, the heartbreak
- **Digital nomads**: Hopping between cities, logging each place and state of mind
- **Long-termists**: Wanting a system that lasts 30 years, not the latest productivity fad
- **Privacy-first types**: Believing your life details shouldn't become training data for cloud providers
- **Futurists**: Believing today's records are raw material for tomorrow's technologies that don't yet exist
- **Every ordinary person**: Who believes their life fragments are worth preserving, even when they're not "productive"

<details>
<summary>🔍 Troubleshooting (Click to expand)</summary>

<br>

**Skill trigger is unreliable**  
→ Use `"/life-index" + intent keyword` in a single message (e.g., `/life-index write journal: ...`)

**Tool error (ModuleNotFoundError)**  
→ Make sure you're using `.venv/bin/python` (not system python), and running from the skill root directory

**Health shows "degraded" on a fresh install**  
→ If you haven't run `life-index index` yet, this is expected. Initialize the index first, then re-run health.

**Journals not written to ~/Documents/Life-Index/**  
→ Run `.venv/bin/python -m tools.query_weather --location "Beijing,China"` from the skill root to verify tools work

**Windows: `write --data '{...}'` is hard to escape**  
→ Use `life-index write --data @first-entry.json` instead (this file is auto-generated by the Agent during onboarding — not shipped with the repo; create your own JSON file if running manually)

**Semantic search unavailable**  
→ Run `.venv/bin/life-index health` to check if fastembed is installed

**venv corrupted (after Python upgrade or system migration)**  
→ Delete the `.venv` directory, then re-run `python3 -m venv .venv && .venv/bin/pip install -e .`

**Upgrading to a new version**  
→ See `docs/UPGRADE.md` (covers version semantics and compatibility commitments)

**Installation health check**  
→ Run `.venv/bin/life-index health` to view the status of all checks

**Permission errors**  
→ Windows: Run terminal as Administrator | Linux/macOS: `chmod -R 755 ~/Documents/Life-Index`

</details>

# 🛠️ Under the Hood

## 📝 Core Feature: Life Journaling

Tell your Agent what you want to record, and it archives it for you:

> /life-index write journal: today I saw old photos of Toto, and suddenly I missed that 2-year-old Diaper Hero — that bittersweet feeling.

The Agent automatically organizes metadata (time, location, people, emotions, tags), generates a structured Markdown file, and stores it on your own hard drive.

> **💡 Trigger tip**: In the current LLM/Agent ecosystem, pure natural-language skill triggering can be unreliable. In practice, putting `/life-index` + an intent keyword (e.g., "write journal", "search logs") in a single message yields a much higher success rate.

## 📖 Core Feature: Life Recall

Three-layer tagging + full-text search + semantic search let you find your past self at any moment:

| Dimension | Examples |
|:---:|:---|
| **Topic** | `think` (Reflection), `create` (Creation), `relation` (Relationships) |
| **Project** | `LifeIndex` (This project), `Parenting` (Fatherhood journey) |
| **Tag** | `Parenthood`, `Memory`, `Bittersweet`, `Diaper Hero` |

Want to find "all bittersweet memories about Toto"? Search `Toto` + `Bittersweet`. You can even search "missing my daughter" in English and find Chinese entries — semantic search supports 50+ languages for cross-language understanding.

### How Does It Find Anything?

The Agent doesn't need to read all 2,000 of your journals — a **dual-pipeline parallel retrieval architecture** lets it zero in on **only the handful it actually needs**:

> **Keyword Pipeline** (exact match) ∥ **Semantic Pipeline** (meaning match) → RRF fusion ranking

Two pipelines run in parallel, results intelligently fused. Brute-force reading 2,000 journals costs ~3 million tokens; after retrieval, only ~5,000 — **99.8% savings**.

<details>
<summary>🔍 Dual-Pipeline Architecture Details (Click to expand)</summary>

```
                    User Query
                   ┌────┴────┐
            ┌──────▼──────┐  ┌──────▼──────┐
            │ Pipeline A  │  │ Pipeline B  │
            │  Keyword    │  │  Semantic   │
            │             │  │             │
            │ L1 Index    │  │ Vector      │
            │ L2 Metadata │  │ Similarity  │
            │ L3 FTS5     │  │ (Multilingual)│
            └──────┬──────┘  └──────┬──────┘
                   └────┬────┘
              RRF Fusion (k=60)
                      │
                  Final Results
```

**Pipeline A — Keyword Matching (finds "what was written")**

| Layer | Data Source | Returns | Purpose |
|:---:|:---:|:---:|:---|
| **L1 Index** | `by-topic/` index files | Date + title + path (~80 bytes/entry) | Quickly narrow candidates by topic/project/tag |
| **L2 Metadata** | YAML Frontmatter (SQLite cache) | All metadata (~500 bytes/entry) | Multi-dimensional filtering by date, mood, people |
| **L3 Content** | FTS5 full-text index | Matching snippets + context (~300 bytes/match) | Exact keyword matching with BM25 ranking |

**Pipeline B — Semantic Search (finds "what was meant")**

| Data Source | Returns | Purpose |
|:---:|:---:|:---|
| Multilingual vector embeddings | Path + similarity (~100 bytes/result) | Finds journals with similar meaning but different keywords; supports cross-language retrieval |

**Both pipelines run in parallel**, with results fused via [Reciprocal Rank Fusion](https://plg.uwaterloo.ca/~gvcormac/cormacksigir09-rrf.pdf) (RRF, k=60) — catching both exact matches and semantically related results.

</details>

## 🛡️ Your Data, Forever Yours

Life Index follows a "local-first" and "data–program separation" strategy:

- **Skill code** lives in your Agent's skills directory
- **Your journals, attachments, and indexes** are stored independently in your system user directory

```
~/Documents/Life-Index/
├── Journals/                    # Main journal directory (by year/month)
│   └── 2026/03/
│       └── life-index_2026-03-04_002.md
├── attachments/                 # Attachments (photos, videos, voice memos)
│   └── 2026/03/
└── by-topic/                    # Auto-generated indexes
    ├── topic_think.md
    ├── project_LifeIndex.md
    └── tag_Parenthood.md
```

**This means**: 20 years from now, even if Life Index ceases to exist, your data will still be there — pure Markdown + YAML, readable by any text editor, clear and timeless.

### ⚠️ Important: Data Backup

- Ask your Agent to **regularly back up the journal data directory with encryption**
- You may choose between **local device** or **cloud** backup

> **Life Index strongly recommends local backup** — protect your **Relic**. Don't voluntarily feed your **Engram** into big tech's **Mikoshi**.
