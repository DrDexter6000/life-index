# RFC-2026-05-20: 继承者作为第二产品对象

## §1 决策

乐乐（主理人女儿）正式列为 Life Index 第二产品对象，与主理人享同等设计优先级。她是 30-50 年后这份档案的翻阅者。

## §2 动机

Life Index 的使命是"遗言式记录"，不止 productivity tool；继承者是这份使命的真正终点。

## §3 影响清单

**成功标准**：从"录入者能查"扩展到"继承者能读、能理解、能听到爸爸的声音"。

**数据模型**：facts / takes / affect 类标注从 nice-to-have 升为路线图必经。

**synthesis 形态**：派生视图、原文优先、声音保留从可选升为硬约束。

**50 年可读性**：不只是 readable，是 meaningful readable。

## §4 本 RFC 不做的事

不修改 CHARTER；不立即新增字段或模块；不指定具体技术实现。本 RFC 是产品决策声明，影响后续 RFC / M16 / M24 设计。

## §5 后续动作锚点

Freeze v1 完成后，本 RFC 内容并入 CHARTER §1.1 扩展节。
