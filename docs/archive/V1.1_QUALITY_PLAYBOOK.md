## Phase 1：修复 search_index.py SSOT 违规

### 目标
消除 `tools/lib/search_index.py` 中手工解析 frontmatter 的代码，统一使用 `lib/frontmatter.py` 的 `parse_journal_file()`。

### 问题根因
第 71-126 行的 `parse_journal()` 函数使用 `line.split(":", 1)` 逐行解析 YAML frontmatter，而项目已有基于 `yaml.safe_load` 的 SSOT 实现（`lib/frontmatter.py`）。

手工解析导致的已知缺陷：
- 含冒号的值被截断：`location: "Lagos, Nigeria: West Africa"` → `value = '"Lagos, Nigeria'`
- 多行 YAML 字符串（`|` 或 `>`）完全丢失
- JSON 数组内含冒号时解析出错
- 未来 schema 变更时此处不会跟着更新

### 操作步骤

1. **定位文件**: `tools/lib/search_index.py`, 第 71-126 行

2. **当前代码**（要替换的部分）:
   ```python
   def parse_journal(file_path: Path) -> Optional[Dict[str, Any]]:
       """解析日志文件，提取可索引内容"""
       try:
           content = file_path.read_text(encoding="utf-8")
           if not content.startswith("---"):
               return None
           parts = content.split("---", 2)
           if len(parts) < 3:
               return None
           fm_text = parts[1].strip()
           body = parts[2].strip()
           # 手工逐行解析 frontmatter（问题根源）
           metadata = {}
           for line in fm_text.split("\n"):
               line = line.strip()
               if ":" in line and not line.startswith("#"):
                   key, value = line.split(":", 1)
                   # ... 后续处理
   ```

3. **替换为**:
   ```python
   def parse_journal(file_path: Path) -> Optional[Dict[str, Any]]:
       """解析日志文件，提取可索引内容"""
       try:
           from .frontmatter import parse_journal_file

           result = parse_journal_file(file_path)
           if "_error" in result:
               return None

           body = result.get("_body", "")
           metadata = {k: v for k, v in result.items() if not k.startswith("_")}

           # 构建可索引文档（保持与 FTS5 表结构兼容）
           doc = {
               "path": str(file_path.relative_to(USER_DATA_DIR)).replace("\\", "/"),
               "title": str(metadata.get("title", "")),
               "content": body,
               "date": str(metadata.get("date", ""))[:10],
               "location": str(metadata.get("location", "")),
               "weather": str(metadata.get("weather", "")),
               "topic": _normalize_to_str(metadata.get("topic")),
               "project": str(metadata.get("project", "")),
               "tags": _normalize_to_str(metadata.get("tags")),
               "file_hash": get_file_hash(file_path),
               "modified_time": datetime.fromtimestamp(
                   file_path.stat().st_mtime
               ).isoformat(),
           }
           return doc

       except (OSError, IOError, ValueError) as e:
           print(f"Warning: Failed to parse {file_path}: {e}")
           return None
   ```

4. **注意事项**:
   - `_normalize_to_str()` 函数（第 129-135 行）保持不变，仍然需要
   - `get_file_hash()` 函数（第 25-31 行）保持不变
   - `from .frontmatter import parse_journal_file` 放在函数内部，避免循环导入
   - `str()` 包裹每个字段值，因为 `yaml.safe_load` 可能返回非字符串类型（如 `date` 返回 `datetime` 对象），而 FTS5 要求字符串。但 `lib/frontmatter.py:69-73` 已经将 datetime 转为 isoformat 字符串，所以 `str()` 是防御性措施


## Phase 3：weather.py — subprocess → 直接 import 调用

### 目标
将 `write_journal/weather.py` 中通过 `subprocess.run()` 调用 `query_weather` 模块的方式，改为直接 Python import 调用，消除每次写日志时冷启动新 Python 进程的开销。

### 问题根因
`tools/write_journal/weather.py` 第 80-99 行：

```python
cmd = [sys.executable, "-m", "tools.query_weather", "--location", location]
proc = subprocess.run(cmd, capture_output=True, text=True, timeout=15)
output = json.loads(proc.stdout)
```

每次写入日志时，如果用户未提供天气信息，`write_journal` 会通过 `subprocess` 启动一个全新的 Python 进程来查询天气。然后解析 stdout 的 JSON 输出。

问题：
- **性能损耗**: 每次额外启动 Python 解释器（~200-500ms 冷启动）
- **错误处理脆弱**: 任何 stderr 输出（warning、log）都可能干扰 JSON 解析
- **可测试性差**: subprocess 调用需要 mock 整个进程，不如函数调用易测
- **已验证**: `from tools.query_weather import query_weather, geocode_location` 可以正常 import

### 操作步骤

1. **定位文件**: `tools/write_journal/weather.py`

2. **保留** `normalize_location()` 函数（第 13-66 行）— 它是纯逻辑，没有问题

3. **重写** `query_weather_for_location()` 函数（第 69-133 行）:

   ```python
   def query_weather_for_location(location: str, date_str: str = "") -> str:
       """
       调用 query_weather 模块获取天气信息

       Args:
           location: 地点名称（英文格式，如 "Chongqing, China"）
           date_str: 日期字符串（可选，格式 YYYY-MM-DD 或更长）

       Returns:
           天气描述字符串（包含温度），失败返回空字符串
       """
       try:
           from ..query_weather import query_weather, geocode_location

           # 地理编码：地点名称 → 经纬度
           geo_result = geocode_location(location)
           if geo_result is None or "error" in geo_result:
               return ""

           latitude = geo_result["latitude"]
           longitude = geo_result["longitude"]

           # 查询天气
           date_part = date_str[:10] if date_str else ""
           if not date_part:
               from datetime import datetime
               date_part = datetime.now().strftime("%Y-%m-%d")

           result = query_weather(latitude, longitude, date_part)

           if not result.get("success"):
               return ""

           # 提取天气描述（与原逻辑保持一致）
           weather_data = result.get("weather", {})
           if isinstance(weather_data, dict):
               description = weather_data.get("description", "")
               simple = weather_data.get("simple", "")
               temp_max = weather_data.get("temperature_max")
               temp_min = weather_data.get("temperature_min")

               parts = []
               if description:
                   parts.append(description)
               elif simple:
                   parts.append(simple)

               if temp_max is not None and temp_min is not None:
                   parts.append(f"{temp_max}°C/{temp_min}°C")
               elif temp_max is not None:
                   parts.append(f"{temp_max}°C")

               return " ".join(parts) if parts else ""
           elif isinstance(weather_data, str):
               return weather_data

           return ""
       except (ImportError, KeyError, ValueError, TypeError) as e:
           return ""
   ```

4. **清理无用导入**: 文件顶部移除不再需要的 import:
   ```python
   # 删除这三行
   import json
   import subprocess
   import sys
   ```
   保留：
   ```python
   from pathlib import Path  # 如果其他地方仍使用，否则也删除
   ```
   **注意**: 检查 `normalize_location()` 是否使用了这些导入。实际检查结果：它不使用 `json`/`subprocess`/`sys`/`Path`，所以可以全部删除。

## Phase 5：统一 frontmatter 格式化

### 目标
评估并统一 `write_journal/frontmatter.py`（92行）和 `lib/frontmatter.py`（348行），消除两套 frontmatter 格式化逻辑并存的局面。

### 问题根因
项目存在两个 frontmatter 格式化实现：

| 文件 | 用途 | 实现方式 |
|:---|:---|:---|
| `tools/lib/frontmatter.py:114-141` | edit_journal 等使用 | `format_frontmatter()` — 按 `FIELD_ORDER` 排序，自动添加 `schema_version` |
| `tools/write_journal/frontmatter.py:12-91` | write_journal 使用 | `format_frontmatter()` — 硬编码字段顺序，无 `schema_version` |

两者的关键差异：
1. **字段顺序**: `lib/` 版本按 `FIELD_ORDER` 列表动态排序；`write_journal/` 版本硬编码字段顺序
2. **schema_version**: `lib/` 版本自动添加；`write_journal/` 版本不添加
3. **空字段处理**: `write_journal/` 版本即使字段为空也输出（如 `weather: ""`）；`lib/` 版本跳过 `None` 值但不跳过空字符串
4. **附件格式化**: `write_journal/` 版本有专门的附件处理逻辑（提取 `rel_path`）；`lib/` 版本将附件作为普通列表处理

### 操作步骤

## Phase 7：index_updater.py — 提取公共索引更新函数

### 目标
消除 `update_topic_index()`、`update_project_index()`、`update_tag_indices()` 三个函数中 ~80% 重复逻辑。

### 问题根因
三个函数都执行相同的流程：获取索引前缀 → 计算 entry → 检查文件 → 追加或创建。唯一差异：前缀名称和值来源。

### 操作步骤

1. **定位文件**: `tools/write_journal/index_updater.py`, 第 20-135 行

2. **添加基础函数**:

   ```python
   def _update_index_file(
       prefix: str, name: str, journal_path: Path, data: Dict[str, Any], header_type: str
   ) -> Optional[Path]:
       """更新单个索引文件的通用逻辑"""
       if not name:
           return None

       BY_TOPIC_DIR.mkdir(parents=True, exist_ok=True)
       index_file = BY_TOPIC_DIR / f"{prefix}{name}.md"

       date_str = data.get("date", "")[:10]
       title = data.get("title", "无标题")
       rel_path = os.path.relpath(journal_path, JOURNALS_DIR.parent).replace("\\", "/")
       entry = f"- [{date_str}] [{title}]({rel_path})"

       if index_file.exists():
           content = index_file.read_text(encoding="utf-8")
           if entry not in content:
               with open(index_file, "a", encoding="utf-8") as f:
                   f.write(entry + "\n")
       else:
           with open(index_file, "w", encoding="utf-8") as f:
               f.write(f"# {header_type}: {name}\n\n")
               f.write(entry + "\n")

       return index_file
   ```
