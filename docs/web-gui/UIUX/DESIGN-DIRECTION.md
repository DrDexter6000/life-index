## 1. 北极星：数字遗产容器

### 1.1 Life Index 不是什么

Life Index **不是**日记工具、不是笔记应用、不是效率软件。  
它是一个 **Agent-native 的数字人生档案馆**——本地优先、纯文本、永不过时。

这意味着 UI 设计的第一目标不是"好看"或"好用"，而是：

> **让用户感受到：这里保存的东西很珍贵，它们安全地留在我自己的电脑上，20 年后我的女儿依然能打开它们。**

### 1.2 核心设计直觉

| 我们要传达的 | 我们不要传达的 |
|---|---|
| 珍贵感 — "这是遗产" | 效率感 — "这是工具" |
| 安全感 — "文件就在我电脑上" | 便捷感 — "随时随地云端同步" |
| 仪式感 — "写下的东西值得被好好保存" | 打卡感 — "保持你的连续记录" |
| 信任感 — "我看得到 Agent 在做什么" | 魔法感 — "AI 在幕后悄悄帮你" |
| 私密感 — "这是我的精神自留地" | 社交感 — "分享给朋友们看看" |

### 1.3 一句话设计方针

> **在深空的安静中，用琥珀的温暖保存人生。**  
> 骨架安静，灵魂独特。核心功能上克制，仪式时刻上爆发。

## 2. 美学系统：Celestial Editorial

### 2.1 为什么坚持 Celestial Editorial

四方顾问中，三方（Kimi、Gemini、Opus）独立得出同一结论：Celestial Editorial 的设计语言**本身就是安静的**。

Kimi 的重新定义最精准——它不是"戏剧化（theatrical）"，而是 **"仪式化（ritual）"**：

| 元素 | 情感功能 |
|---|---|
| 深空背景 (#0A0E14) | 创造夜间写作的"安全壳"——离开嘈杂互联网，进入个人精神自留地 |
| 琥珀金强调 (#FFE792) | 温暖光源——台灯、烛光的联想，不是舞台灯光 |
| 衬线标题 (Newsreader) | 信纸、日记本的联想——手写的温度 |
| 无衬线正文 (Manrope) | Agent/技术侧的清晰感——人与机器的对话 |
| 有机圆角 (3rem / full rounded) | 减少数字产品的冰冷感——一切柔和 |

**裁定**：Celestial Editorial 色板和字体组合**全局贯穿**所有页面。不需要在任何页面"降级"到白底极简。控制的是**装饰密度**，不是色板本身。

### 2.2 色板定义

#### 基底

| Token | Hex | 用途 |
|---|---|---|
| `background` | #0A0E14 | 全局画布——深邃的青黑色，提供无限深度 |
| `surface` | #12161E | 主要内容区域背景 |
| `surface-container-low` | #1A1E28 | 次级容器 |
| `surface-container` | #1E2330 | 标准容器 |
| `surface-container-high` | #232836 | 浮动元素背景 |
| `surface-bright` | #262C36 | 活跃/高亮容器 |

#### 强调色

| Token | Hex | 用途 |
|---|---|---|
| `primary` | #FFE792 | 核心强调——主要操作、活跃状态、关键词高亮 |
| `primary-container` | #FFD709 | 主要操作悬停/按压态 |
| `secondary` | #F9873E | 温暖辅助——交互反馈 |
| `tertiary` | #85FFF2 | 冷调辅助——语义搜索匹配标注、次要数据 |
| `on-surface` | #F9F9FF | 主要文字——高可读性的奶白色 |
| `on-surface-variant` | #C4C6CF | 次要文字——元数据、占位符 |
| `error` | #FF716C | 错误状态——仅文字和微光，不把整个容器变红 |

#### 视觉规则

- **No-Line 规则**：禁止 1px 实线边框用于分区。所有结构边界通过背景色阶差异定义
- **Ghost Border**：如果容器确实需要定义，使用 `outline-variant` (#44484F) + 15% opacity
- **Glassmorphism**：浮动元素（modal、tooltip、导航）使用半透明 `surface` + `backdrop-blur: 12-16px`。注意：不追求极端 blur 值，优先保证低端设备流畅度
- **无纯灰**：所有中性色必须带有青色（surface 系列）或琥珀色调（on-surface 系列），禁止 #808080

### 2.3 字体系统

| 角色 | 字体 | 场景 |
|---|---|---|
| Display / Headlines | **Newsreader** (Serif) | 日志标题、页面大标题、仪式时刻的文字——信纸的温度 |
| Body / Labels / UI | **Manrope** (Sans-serif) | 正文、元数据、按钮、输入框——技术的清晰 |

**排版规则**：
- 标题用衬线领叙事，正文用无衬线做功能——这个节奏不可打破
- Display 与 Body 之间保持高对比度字号差（display-lg: 3.5rem vs body-lg: 1rem）
- 中文内容 fallback 至系统默认衬线/无衬线字体，确保正确渲染

### 2.4 圆角与形状

**分层圆角策略**：

| 元素类型 | 圆角规格 | 说明 |
|:---|:---|:---|
| **大视觉元素**（卡片、容器、区块） | `rounded-xl` (1.5rem) 至 `rounded-2xl` (2rem) | 保持有机柔和，符合 Celestial 美学 |
| **小视觉元素**（按钮、输入框、下拉菜单、chip、toggle） | `rounded-md` (0.375rem) 至 `rounded-lg` (0.5rem) | **比原先减小约50%**，避免过度圆润显得不自然 |

**约束**：
- 禁止 `rounded-none` 和 `rounded-sm`——一切保持柔和
- 禁止小元素使用 `rounded-full` pill 形状（输入框、按钮等改为中等圆角）

### 2.5 间距与呼吸

- 主要区块间距：`spacing-16` (4rem) 以上——让界面"呼吸"
- 列表项间距：`spacing-6` (1.5rem) 至 `spacing-8` (2rem)，禁止分割线
- 侧边距充足——创造"纸张"感，内容不贴边

## 3. 交互模态：密度分层

这是本文档最核心的架构决策。来自 Gemini 的"状态隔离"提案，由 Opus 细化为三档。

### 3.1 三档模态

| 模态 | 触发条件 | 装饰密度 | 目标感受 |
|---|---|---|---|
| **沉浸输入态** (Zen Typing) | 用户光标激活写作区域 | **极低** | 比 Calmly Writer 还安静——但底色是深空，不是白板 |
| **浏览交互态** (Archivist Mode) | Dashboard、搜索结果、日志阅读 | **中等** | Celestial Storybook 的标准呈现 |
| **仪式时刻** (Ritual Moments) | 完成写作、On This Day、年度摘要 | **集中爆发** | Florence 关键帧——精准的情感触发 |

### 3.2 沉浸输入态详述

当用户开始写作时，界面进入最克制的状态：

```
进入条件：光标聚焦到写作区域
视觉变化：
  ├── 背景保持 #0A0E14（不切换到白色）
  ├── 顶部导航栏 → fade out（0.3s ease）
  ├── 侧边元素 → fade out
  ├── Glassmorphism 浮动元素 → 透明度降至 0
  ├── 所有 glow / ambient 装饰 → 淡出
  └── 屏幕上只剩：文字、光标、底部微弱的保存状态
退出条件：光标离开写作区域 / 用户按 Esc / 鼠标移动到屏幕边缘
```

### 3.3 浏览交互态详述

标准的 Celestial Editorial 呈现：

```
视觉元素：
  ├── Tonal layering 全量（surface 色阶堆叠）
  ├── Glassmorphism 浮动面板（导航、搜索框）
  ├── 琥珀金强调色活跃（按钮、高亮、活跃标签）
  ├── 卡片 hover 时柔和 glow（不是 pop up，是微光扩散）
  └── 大量留白（Ma / 間）
密度控制：
  ├── 每屏信息量不超过 3 个主要区块
  ├── 不堆砌统计数字
  └── 不使用 1px 分割线
```

### 3.4 仪式时刻详述

正反馈集中在少数情感高点，而非均匀撒在每个页面：

| 时刻 | 交互 | 实现复杂度 |
|---|---|---|
| **写作完成** | 可选的"封存"过渡——内容区域柔和收束 + 右下角 "已安全保存到本地" toast | CSS transition |
| **On This Day** | 打开时的渐显过渡——旧日志从深处"浮现"，带微弱的琥珀光晕 | CSS animation |
| **里程碑** | 达到特定篇数时的一次性微交互——不是烟花，是一个安静的光点扩散 | CSS animation |

**严格禁止**：
- 打字音效
- 烟花/粒子特效
- 任何"表演性"反馈
- Streaks / 积分 / 徽章

## 4. 页面级设计规范

### 4.1 Dashboard

**模态**：浏览交互态

**布局**：
```
┌─────────────────────────────────────────────┐
│  导航栏 (Glassmorphism)                       │
├─────────────────────────────────────────────┤
│                                             │
│  ┌─ On This Day ─────────────────────────┐  │
│  │  (如果今天有历史日志，优先展示)          │  │
│  │  衬线标题 + 日期 + 摘要片段             │  │
│  └───────────────────────────────────────┘  │
│                                             │
│  ┌─ 关键数字 ─────────────────────────────┐  │
│  │  总篇数 · 最近一篇日期 · 本月记录数     │  │
│  │  （三个数字，不多不少）                  │  │
│  └───────────────────────────────────────┘  │
│                                             │
│  ┌─ 时间线 ──────────────────────────────┐  │
│  │  暗色背景上的发光节点                   │  │
│  │  节点亮度/大小 ∝ 内容丰富度             │  │
│  │  点击节点 → 跳转到对应日志              │  │
│  └───────────────────────────────────────┘  │
│                                             │
│  ┌─ Agent Activity (低调) ───────────────┐  │
│  │  最近的索引更新 / 天气查询 / 语义索引    │  │
│  │  "2 小时前 · 更新了语义索引 (+3 篇)"    │  │
│  └───────────────────────────────────────┘  │
│                                             │
└─────────────────────────────────────────────┘
```

**设计要点**：
- ❌ 不做仪表盘式的统计堆砌（"Aura Stability" 等抽象指标）
- ❌ 不做 D3.js 花园/星座图
- ✅ On This Day 是情感入口——如果没有历史日志则不显示，不留空白占位
- ✅ Agent Activity 栏建立信任感——用户看到 Agent 在默默工作
- ✅ 时间线用 CSS 实现（发光节点 = `box-shadow` + `radial-gradient`），不用 Canvas/WebGL

### 4.2 写作页面

**模态**：浏览交互态 → 沉浸输入态（光标激活后过渡）

**结构**：
```
写作页面：
├── 标题输入区
│   ├── 巨大的 Newsreader 衬线字体
│   ├── 占位符："给这篇记录起个名字..."（on-surface-variant 色）
│   └── 建立仪式感——这是"开始书写"的信号
├── 正文输入区
│   ├── Manrope 无衬线字体
│   ├── 行高 1.8（generous spacing）
│   ├── 段落间距明显
│   ├── 侧边距充足（创造"纸张"感）
│   └── 光标激活后 → 进入沉浸输入态
├── 工具栏
│   ├── 默认隐藏（鼠标悬停在顶部边缘时显示）
│   ├── 仅保留：加粗、斜体、列表、图片上传、标签
│   └── 标签输入：# 触发自动补全
├── 元数据区
│   ├── 日期：自动捕获，可编辑
│   ├── 地点 / 天气：灰字提示"如不填写，将使用默认地点并查询天气数据"
│   ├── 心情：可选预设（文字标签，不是 emoji 选择器）
│   ├── 话题：work/learn/health 等快捷选择（chip 形式）
│   └── 核心规则：用户填了 → 以用户为准；没填 → AI 自动提炼
├── 附件区
│   ├── 拖拽上传 / 点击上传
│   ├── 支持粘贴 URL（自动下载归档）
│   └── 附件预览（缩略图）
└── 保存反馈
    ├── 自动保存 → 右下角微弱 toast "已保存到本地"
    └── 手动提交 → 可选封存过渡动画
```

**关键交互**：
- 光标呼吸：光标周围极微弱的琥珀色脉动（CSS `animation: pulse`），不是闪烁，是"呼吸"——2s 周期，opacity 0.1-0.3
- 保存确认：右下角 toast，`on-surface-variant` 色，2 秒后自动消失
- 封存仪式：写完手动提交后，内容区域柔和收束（`transform: scale(0.98)` + 亮度微降 → 恢复），暗示"它被安全地收起来了"

### 4.3 搜索页面

**模态**：浏览交互态

**结构**：
```
搜索页面：
├── 搜索框
│   ├── pill 形状（rounded-full）
│   ├── surface-container-lowest 背景
│   └── 聚焦时琥珀色微光（不是粗边框）
├── 搜索过程可视化（差异化功能）
│   ├── 双管道状态：关键词管道 ∥ 语义管道
│   ├── 各管道进度用微弱的动画指示
│   └── "关键词匹配: 12 篇 · 语义相关: 8 篇 · RRF 融合: 15 篇"
├── 结果列表
│   ├── 卡片间距 spacing-6，无分割线
│   ├── 卡片用 tonal layering 区分层级（surface → surface-container）
│   ├── 关键词高亮：primary (#FFE792)
│   ├── 语义匹配标注：tertiary (#85FFF2)
│   ├── 每张卡片显示：标题 + 日期 + 摘要片段 + 匹配类型
│   └── hover 时柔和 glow（不是 pop）
└── 筛选器（可选展开）
    ├── 话题筛选：chip 形式
    ├── 日期范围
    └── 心情筛选
```

**差异化要点**：
- 搜索过程双管道可视化是**所有竞品都没有的**——它同时体现了 Agent 透明度和技术独特性
- 不要隐藏搜索过程，让用户看到"你的 Agent 正在同时用两种方式帮你找"

### 4.4 阅读页面

**模态**：浏览交互态

**布局**：
```
阅读页面：
├── 标题区
│   ├── Newsreader 衬线，display 级字号
│   ├── 日期 + 地点 + 天气（on-surface-variant）
│   └── 心情标签 + 话题标签（chip 形式）
├── 正文区
│   ├── Manrope 无衬线
│   ├── 行高 1.8，段距明显
│   ├── 最大宽度 65ch（最佳阅读行宽）
│   └── 附件以内联缩略图 / 链接形式展示
├── 元数据侧栏（宽屏时显示，窄屏折叠）
│   ├── 文件路径（数据主权可视化）
│   ├── 创建时间 / 修改时间
│   ├── 相关日志推荐（语义关联）
│   └── Agent 处理记录（"标签由 AI 提炼 · 天气由 API 查询"）
└── 操作区
    ├── 编辑按钮 → 跳转编辑页面
    └── 导出（Markdown 原文）
```

**数据主权可视化**：
- 侧栏显示完整文件路径：`~/Documents/Life-Index/Journals/2026/03/life-index_2026-03-07_001.md`
- 这不是技术细节，这是**信任建设**——用户看到文件就在自己电脑上

### 4.5 编辑页面

**模态**：浏览交互态 → 沉浸输入态（与写作页面共享交互模式）

- 复用写作页面的所有设计规范
- 区别：预填充已有内容，标题区显示"编辑中"状态标识
- 元数据字段全部可编辑

## 5. 差异化武器

按优先级排序。这是 Life Index 与所有竞品的根本区分，**必须在 UI 中可感知**。

### P0 — 数据主权的可感知性

| 设计决策 | 实现方式 |
|---|---|
| 文件路径在阅读页面可见 | 元数据侧栏显示完整本地路径 |
| "保存到本地" 提示 | 写作页面的保存 toast 强调"本地" |
| 无云端/同步 UI 元素 | 不出现任何 "sync"、"cloud"、"backup" 图标 |

> 竞品对比：Day One/Clearful 说"加密"但数据在云端；Reflect 说"本地"但没有文件级可见性。Life Index 让你**直接看到文件在哪里**。

### P1 — Agent 工作透明度

| 设计决策 | 实现方式 |
|---|---|
| Dashboard Agent Activity 栏 | 显示最近的索引更新、天气查询等操作日志 |
| 搜索双管道可视化 | 实时显示关键词管道 / 语义管道的工作状态 |
| 阅读页 Agent 处理记录 | 标注哪些元数据由 AI 提炼、哪些由用户手填 |

> 竞品对比：无竞品做到 Agent 工作透明度。这是 Life Index 独有的设计语言。

### P2 — 时间维度交互

| 设计决策 | 实现方式 |
|---|---|
| On This Day | Dashboard 情感入口，历史日志渐显过渡 |
| 时间线导航 | Dashboard 发光节点时间线，可点击跳转 |
| 主题交叉视图（v3） | 时间 × 话题矩阵，展示人生的多维切面 |

> 竞品对比：Day One 有 On This Day 但没有主题交叉；Clearful 有 Rediscover 但无语义搜索维度。

### P3 — 仪式感微交互

| 设计决策 | 实现方式 |
|---|---|
| 光标呼吸 | CSS animation，极微弱琥珀色脉动 |
| 保存确认 | toast，强调"本地" |
| 封存仪式 | 提交后内容区域柔和收束 |
| On This Day 渐显 | 旧日志从深处浮现 |

## 6. 分阶段计划

### v1 — "能用且有性格"

**目标**：Celestial Editorial 全局落地 + 核心功能可用

- [ ] Celestial 色板和字体全局应用

## 7. 明确拒绝清单

| 拒绝项 | 理由 |
|---|---|
| ❌ 极简白板风格（Day One/Pencake 式） | 红海竞争，无法体现品牌定位，且无法在原生 App 体验上胜出 |
| ❌ D3.js / Canvas 花园/星座图可视化 | 维护成本与收益不成比例，v1-v3 均不考虑 |
| ❌ WebGL / 粒子特效 / 重度动画 | 低端设备性能风险，CSS transition/animation 完全足够 |
| ❌ Streaks / 积分 / 徽章 / 打卡机制 | 遗产不需要打卡，制造焦虑而非安全感 |
| ❌ 社交功能（分享、评论、协作） | 从根本上破坏私密感 |
| ❌ 情绪天气隐喻（用天气图标代表情绪趋势） | 认知成本高，直接显示心情标签更诚实 |
| ❌ 过度抽象的指标（"Aura Stability"、"Cosmic Joy"） | 用户看不懂，用具体数字（"本月 12 篇"）代替 |
| ❌ 纯黑 #000000 用于非最深层容器 | 所有深色必须带青色调，保持 Celestial 一致性 |
| ❌ 1px 实线边框用于分区 | No-Line 规则——用色阶差异定义边界 |
| ❌ 打字音效 | 不增加非视觉感官刺激 |

## 8. 技术约束与注意事项

### 8.1 性能底线

- `backdrop-blur` 值不超过 16px——20px 在低端设备上可能掉帧
- 所有动画使用 `transform` 和 `opacity`（GPU 加速属性），不动画 `width`/`height`/`margin`
- 字体加载策略：`font-display: swap`，衬线字体文件需 subset 以减小体积
- 仪式时刻动画总时长不超过 1.5s——用户不应该"等待"仪式结束

### 8.2 可访问性底线

- 所有文字与背景的对比度达到 **WCAG AA 标准**（4.5:1 正文，3:1 大字）
- 深色模式下 `on-surface` (#F9F9FF) 对 `background` (#0A0E14) 的对比度 ≈ 18.5:1 ✓
- `on-surface-variant` (#C4C6CF) 对 `surface` (#12161E) 需验证并调整至达标
- 所有交互元素有清晰的 focus 状态（琥珀色外轮廓，不依赖颜色传达信息）

### 8.3 响应式策略

- 桌面优先（Life Index 的核心使用场景是电脑前的深度写作）
- 移动端：保持色板和字体，简化布局（单列、侧栏折叠、时间线水平滚动）
- 有机布局中的不对称元素在移动端回归对齐——不强求小屏上的"collage"效果
