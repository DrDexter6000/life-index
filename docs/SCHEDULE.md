# Life Index 定时任务指南

> 本文档指导 Agent 设置和执行 Life Index 定时任务
> **版本/状态**: 详见 [README.md](./README.md)

---

## 前置说明

**重要约束**：
- 必须使用 Agent 自带的定时任务功能，禁止使用OS操作系统级定时任务（如 Windows Task Scheduler）
- 所有时间均为用户本地时区
- "用户日志"指 Life Index 系统记录的日志，不包含系统运行日志或 Agent 自身日志

---

## 定时任务清单

### 任务1：Life Index 日报

**执行时间**：每天 22:00

**任务目标**：生成今日日志的智能日报并推送给用户

**执行步骤**：

1. **获取今日日志**
   - 检索当天所有用户日志
   - 如无日志，跳过本次执行

2. **生成报告内容**
   - 汇总归纳：整合今日所有日志的核心内容
   - 要点提炼：提取 3-5 个关键事项
   - AI 建议：基于今日日志内容给出个性化建议
   - 要事提醒：识别需要跟进的事项

3. **推送消息**
   - 推送到主会话和已配置的即时通讯渠道
   - 不生成文件，仅推送

---

### 任务2：Life Index 周报

**执行时间**：每周日 22:10

**任务目标**：生成本周日志的智能周报并推送给用户，同时更新年度摘要

**执行步骤**：

1. **获取本周日志**
   - 时间范围：本周一 00:00 至当前
   - 如无日志，跳过本次执行

2. **生成报告内容**
   - 汇总归纳：整合本周所有日志的核心内容
   - 要点提炼：提取 5-7 个关键事项
   - AI 建议：基于本周日志内容给出个性化建议
   - 要事提醒：识别需要跟进的事项

3. **获取国际热点**
   - 检索本周国际热点新闻和社交媒体话题
   - 筛选 Top 5 热点事件
   - 简要描述每个热点事件

4. **更新年度摘要**
   - 调用 generate_abstract 工具，参数为当前年份
   - 静默执行，失败不影响周报推送

5. **推送消息**
   - 推送到主会话和已配置的即时通讯渠道
   - 不生成文件，仅推送

---

### 任务3：Life Index 月报

**执行时间**：每月最后一天 18:30

**任务目标**：生成本月日志的智能月报，保存文件并推送给用户

**执行步骤**：

1. **获取本月日志**
   - 时间范围：本月1日 00:00 至当前
   - 如无日志，跳过本次执行

2. **生成报告内容**
   - 汇总归纳：整合本月所有日志的核心内容
   - 要点提炼：提取 7-10 个关键事项
   - 要事提醒：识别需要跟进的事项
   - AI 甄选：从本月日志中推荐最重要的 5 篇，简述推荐理由

3. **获取国际热点**
   - 检索本月国际热点新闻和社交媒体话题
   - 筛选 Top 10 热点事件
   - 简要描述每个热点事件

4. **保存文件**
   - 文件位置：用户数据目录下的 Journals/YYYY/MM/
   - 文件命名：monthly_report_YYYY-MM.md
   - 格式：Markdown，包含完整的报告内容

5. **推送消息**
   - 推送到主会话和已配置的即时通讯渠道
   - 消息中包含文件路径

### 任务5：向量索引增量更新

**执行时间**：每天 23:50

**任务目标**：基于今日新增日志更新向量索引

**执行步骤**：

1. **检查今日日志**
   - 确认今天是否有新增日志
   - 如无新增，跳过本次执行

2. **执行增量更新**
   - 调用 build_index 工具
   - 默认模式即为增量更新

3. **记录结果**
   - 更新成功则静默完成
   - 更新失败则记录错误，下次重试

---

### 任务6：向量索引全量重建

**执行时间**：每月1日 03:30

**任务目标**：基于所有日志重建向量索引

**执行步骤**：

1. **执行全量重建**
   - 调用 build_index 工具
   - 使用重建模式

2. **记录结果**
   - 重建成功则静默完成
   - 重建失败则记录错误，通知用户

## 任务设置指南

Agent 在首次加载 Life Index skill 时，应检查是否已设置上述定时任务。如未设置，应主动创建。

**创建任务时的注意事项**：

1. **任务标识**：每个任务使用唯一标识符，如 `life-index-daily-report`
2. **时区处理**：使用用户本地时区，非 UTC
3. **失败重试**：如任务执行失败，不应阻塞后续任务
4. **日志记录**：任务执行结果应记录到系统日志，便于排查问题
5. **用户通知**：任务创建成功后，可选择性通知用户

---

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 无日志数据 | 跳过本次执行，不发送空报告 |
| 日志检索失败 | 记录错误，下次执行时重试 |
| 热点获取失败 | 报告中省略热点部分，继续执行 |
| 文件写入失败 | 先推送内容，再提示用户检查存储空间 |
| 推送失败 | 保存报告内容，待用户上线时展示 |
